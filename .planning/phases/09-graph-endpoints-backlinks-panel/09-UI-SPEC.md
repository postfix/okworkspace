# Phase 9 тАФ UI Design Contract

> Visual and interaction contract for the **"Referenced by" (backlinks) panel** тАФ the only user-facing UI in Phase 9. The graph endpoints (`/api/v1/graph`, `/api/v1/graph/local`) are backend-only; the graph visualization UI is Phase 10 and is explicitly OUT OF SCOPE here.
>
> This is a subsequent phase on a shipped app (v0.9.9) with an established, hand-rolled token design system (`web/src/styles/tokens.css`). This contract REUSES existing tokens and component patterns тАФ it introduces no new design language. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Scope

**In scope (this contract covers only these):**
- The collapsible panel container appended to the BOTTOM of the page read view (`web/src/routes/PageView.tsx`), below the rendered Markdown body тАФ LOCKED by `09-CONTEXT.md`.
- The `Referenced by (N)` section header with an expand/collapse chevron (collapsed + expanded states).
- The list of backlink entries (page title/path, click-to-navigate via the existing page route, hover + keyboard focus).
- The quiet empty state ("No backlinks yet").
- Loading and error states for the react-query fetch of the backlinks endpoint.

**Out of scope (do NOT spec тАФ Phase 10 / later):**
- Graph visualization / canvas / force-directed layout, edge-type toggles, tag nodes, hover-highlight, local-graph panel UI.
- Backlink context snippets (the linking sentence) тАФ deferred to v2 (LINK-F1).
- Right-side dockable panel layout тАФ explicitly rejected for Phase 9 in favor of the below-page collapsible.

## Component & Interaction Contract

### Container тАФ collapsible section (LOCKED placement)
- Rendered at the BOTTOM of `PageView.tsx`, AFTER the `LivePreviewEditor` read surface (and after the empty-body `<p>` branch), still inside the `.pageview` article. Additive only тАФ must not disturb the CM6 read rendering.
- Aligned to the page body column (`--prose-max-width: 720px`, same horizontal alignment as the rendered Markdown).
- Separated from the body above by a single top border (`1px solid var(--color-border)`) plus `--space-lg` top margin.
- The panel is only rendered for an existing, loaded page (i.e. inside the success branch; not on the 404/empty-error branches).

### Header / toggle (collapsed тЖФ expanded)
- A single full-width `<button type="button">` acting as the disclosure control.
- Pattern mirrors the existing tree caret (`web/src/components/LeftTree.tsx` `.tree-caret`): a `lucide-react` chevron тАФ `ChevronRight` when collapsed, `ChevronDown` when expanded (size 16, `aria-hidden`), followed by the label `Referenced by (N)`.
- `aria-expanded={open}` on the button; `aria-label` not required since the visible label is descriptive. The count `N` is the number of backlinks.
- Default state: **expanded** when N тЙе 1 (the backlinks are the value); the user can collapse. Toggle persistence is at the planner's discretion (a zustand boolean is the established pattern) but is NOT required for Phase 9.
- Min hit height `--hit-min-height` (40px); hover wash `--color-hover`; `:focus-visible` accent ring (global).

### Entry list
- A `<ul className="navtree">` (reuses existing list geometry: `--space-xs` gap, no bullets).
- Each entry is a `<li>` containing a full-width `<button type="button" className="navrow navrow-page">` тАФ the SAME row component used by `RecentList` and the left tree, so it is visually identical to existing navigation rows.
- Each row: a small `lucide-react` icon (`FileText` size 16, `className="tree-icon"`, `aria-hidden`) + `<span className="tree-label">{title}</span>`. Title is the linking page's title (fallback to its path), per LINK-02.
- Click тЖТ navigate to the linking page via the EXISTING page route (`navigate(\`/app/page/\${linkingPath}\`)`), identical to `RecentList`.
- Hover: `--color-hover` wash (inherited from `.navrow-page:hover`). Keyboard focus: global accent `:focus-visible` ring. Rows are fully keyboard-operable (real `<button>`s).

### Loading state
- While the backlinks query is `isLoading`: render a single quiet muted line "Loading backlinksтАж" at `--font-size-label`, `--color-text-muted`. Do NOT block or replace the page body; the panel region simply shows the loading line. No spinner is required (matches the app's existing text-status convention, e.g. `.pageview-status` "LoadingтАж").

### Error state
- On query error: render a single quiet muted line "Couldn't load backlinks. Refresh to try again." at `--font-size-label`, `--color-text-muted`. Non-blocking тАФ the page body remains fully readable. No retry button (refresh is the recovery path), consistent with the panel's quiet, additive intent.

### Empty state (LOCKED: quiet)
- When the page has zero backlinks: show the header `Referenced by (0)` followed by a single muted line "No backlinks yet" at `--font-size-label`, `--color-text-muted`. Quiet and unobtrusive per `09-CONTEXT.md` тАФ never a prominent empty-state card or illustration.

## Accessibility

- Disclosure toggle: real `<button>` with `aria-expanded`; chevron icons `aria-hidden`; visible text label is the accessible name.
- Each backlink entry: real `<button>` (keyboard focusable + Enter/Space activatable); icon `aria-hidden`; title text is the accessible name.
- All interactive elements meet `--hit-min-height` (40px) and use the global accent `:focus-visible` ring (2px, 2px offset).
- Count `(N)` is part of the visible label, so screen readers announce it with the header.
