<interface_contract>
Extends the 03-01 typed index WITHOUT changing the result DTO envelope. New result kinds:
- `kind:"attachment"` тЖТ `title` = original filename, `path` = owning page path (from `AttachmentMeta.PagePath`), `page_title` = owning page title, snippet from filename/extracted_text.
- `kind:"heading"` тЖТ `title` = heading text, `path` = page path, `anchor` = "#slug" (GitHub-style), `page_title` = owning page title.

Heading doc ID convention (set in 03-01 doc.go): `pagePath + anchor`. Heading anchor slug: GitHub-style тАФ lowercase, spacesтЖТ`-`, strip punctuation except `-`. The Go scanner's slug MUST equal the id `rehype-slug` assigns to the rendered heading so deep-links resolve.

Document mappings for `heading`/`attachment` are already declared in 03-01 `mapping.go`; this plan populates the fields. The `page_headings(page_path, heading_id)` table from migration 0007 tracks per-page heading ids for stale cleanup.
</interface_contract>

|----------|-------------|
| attachment `.txt`/meta on disk тЖТ index | Extracted text + meta read via safe-path resolver into the derived index |
| page body тЖТ heading docs | Opaque body text scanned for headings (no AST, no re-emit) |
| npm registry тЖТ build | rehype-slug install crosses into the build supply chain |
| rendered heading id тЖФ search anchor | Cross-tier slug equality the deep-link depends on |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-11 | Tampering | attachments.go / headings.go file reads | mitigate | All reads via `repo.Read` (`TxtPath`/`MetaPath`/page paths); never `os.*`; tolerate missing `.txt` without erroring. |
| T-03-12 | Information Disclosure | rebuild/index attachment + heading docs | mitigate | Same trash exclusion as pages; an attachment whose owning page is trashed/deleted is removed via the delete path. |
