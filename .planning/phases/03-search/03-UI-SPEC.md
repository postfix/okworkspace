# Phase 3 — UI Design Contract

> Visual and interaction contract for the Search phase (SRCH-01..06). Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **This phase EXTENDS the Phase 0–2 design system** (`web/src/styles/tokens.css`, `controls.css`, and the established Dialog / LeftTree / AttachmentCard / RoleBadge / ExtractionStatus patterns). It introduces **no new design language** and **no new tokens** — every value below is an existing `var(--…)`. The single net-new surface is the Obsidian-style **⌘K quick-switcher** (command palette), assembled from existing primitives.
>
> **Hidden-Git rule (carried from Phases 0–2):** no version-control vocabulary anywhere in this UI — no "commit", "index", "Bleve", "HEAD", "repo", "Git". The user sees only "search", "results", pages, headings, and attachments.

**Fixed dimensions (existing tokens / media values, not spacing):**

| Value | Usage |
|-------|-------|

## Typography

All four roles are the existing `tokens.css` roles — exactly 4 sizes, exactly 2 weights. No new type.

| Role | Token | Size | Weight | Line Height | Usage in this phase |
|------|-------|------|--------|-------------|---------------------|
| Body | `--font-size-body` | 16px | `--font-weight-regular` (400) | `--line-height-body` (1.5) | Search input text; result title |
| Label | `--font-size-label` | 14px | `--font-weight-regular` (400) | `--line-height-label` (1.4) | Type badge text; result snippet; group label; keyboard-hint footer; "in {page title}" sub-line on heading/attachment results |
| Heading | `--font-size-heading` | 20px | `--font-weight-semibold` (600) | `--line-height-heading` (1.2) | No-results / error panel heading (scaled-down empty-state) |
| Display | `--font-size-display` | 28px | `--font-weight-semibold` (600) | `--line-height-display` (1.2) | Not used inside the palette (reserved by system) |

**Weight rules:**
- Result **title** is `--font-weight-semibold` (600) — the scan anchor (matches `.attachment-card-name` and `.navrow` emphasis).
- Result **snippet**, **group label**, **sub-line**, and **badge text** are `--font-weight-regular` (400).
- **Matched-term highlight** inside titles/snippets uses `--font-weight-semibold` (600) (NOT a background fill, NOT a third color — see Color). This is the only weight change applied to a substring.

## Color

60 / 30 / 10 split, identical to the Phase 0–2 contract. No new colors.

| Role | Token | Value | Usage in this phase |
|------|-------|-------|---------------------|
| Dominant (60%) | `--color-bg` | `#ffffff` | Palette panel background; result-list background |
| Secondary (30%) | `--color-surface` | `#f6f7f8` | Type-badge fill; group-label background; result-row hover (`--color-tree-hover` → `--color-surface`); search-input row separator surface |
| Accent (10%) | `--color-accent` | `#2563eb` | RESERVED — see list below |
| Scrim | (inline `rgba(16,24,40,0.45)` via `.dialog-backdrop`) | — | Modal backdrop behind the palette |
| Muted text | `--color-text-muted` | `#6b7280` | Snippet text, group label, sub-line ("in {page}"), keyboard hints, search icon, placeholder |
| Border | `--color-border` | `#e3e5e8` | Palette border, input border, group/row hairline separators |
| Warning | `--color-warning` | `#d97706` | Inline error row text + border (reuses `.banner-warning`); the only semantic accent here |

**Accent (`--color-accent`) reserved for — explicit list (never "all interactive elements"):**
1. The **keyboard-active result row** — left inset bar `inset 2px 0 0 var(--color-accent)` + `--color-tree-active-bg` fill + accent title text, reusing the EXACT `.navrow-active` treatment.
2. The **search input focus ring** — `:focus-visible` 2px accent outline + accent border (reuses `.input:focus-visible`).
3. The **drop-target / focus ring** on the top-bar trigger (`:focus-visible`).

Accent is NOT used for: type badges (neutral surface chip), matched-term highlight (bold weight only), snippet text, group labels, or the result count. There is **no primary "Search" CTA button** in this phase — search is an instant, keyboard-first overlay with no submit button — so the accent-button slot is intentionally empty.

**Destructive:** none. This phase has **no destructive actions** (read-only search). `--color-destructive` is not used.

**Matched-term highlight rule (explicit):** Bleve returns fragment highlights. Render matched terms with `--font-weight-semibold` (600) and normal `--color-text`. Do **NOT** introduce a yellow/`<mark>` background fill and do **NOT** use `--color-accent` for highlights — both would violate the 10% accent reservation and the no-new-color rule. Sanitize highlight markup: map Bleve's fragment tags to a `<strong>`/`<span class="search-hl">` styled only by weight (no `dangerouslySetInnerHTML` of raw server HTML — consistent with the Phase 1 stored-XSS guard: `rehype-raw` OFF).

## Copywriting Contract

No version-control vocabulary. Plain, Obsidian-familiar, action-oriented copy.

| Element | Copy |
|---------|------|
| Primary CTA | *(none — keyboard-first overlay, no submit button)*. The top-bar trigger label/affordance reads **"Search"** with a `Search` icon and a `⌘K` / `Ctrl K` keycap hint. |
| Search input placeholder | `Search pages, headings, and attachments…` |
| Empty state (palette just opened, no query) — heading | `Search your workspace` |
| Empty state — body | `Find any page, heading, or attachment. Start typing to see results.` |
| Empty state — keyboard hint footer | `↑↓ to navigate · ↵ to open · esc to close` |
| Typing / loading | `Searching…` (with `.spinner`); shown only if a query is in flight > ~150ms (avoid flicker). |
| No-results — heading | `No matches` |
| No-results — body | `Nothing matched "{query}". Try a different word or check the spelling.` |
| Error state — heading | `Search is unavailable` |
| Error state — body | `Something went wrong while searching. Try again in a moment.` (Rendered as a `.banner-warning`-style inline row; never exposes server/index internals.) |
| Result group labels | `Pages` · `Headings` · `Attachments` (plural, sentence-style; shown only for groups that have ≥1 result) |
| Result count (optional, muted) | `{n} results` (omit when 0; singular `1 result`) |
| Heading-result sub-line | `in {page title}` (muted label) |
| Attachment-result sub-line | `in {owning page title}` (muted label — the result links to the owning page, SRCH-05) |
| Destructive confirmation | not applicable — no destructive actions in this phase |

**Copy rules:** Sentence case for body/labels; never "Query", "Index", "Hits", or "Results found in repo". `{query}` is rendered as plain text (escaped) inside quotes. Truncate `{query}` in the no-results line to a reasonable length to avoid layout break.

## Component Contract — ⌘K Quick-Switcher (the one net-new surface)

> Build as a new component (e.g. `web/src/components/search/SearchPalette.tsx` + `SearchPalette.css`) using only the primitives above. State lives in zustand (open/closed) per the locked stack; the query → results fetch uses `@tanstack/react-query` (debounced). Wire the search request through `web/src/api/client.ts` query patterns.

### Overlay & layout
- **Backdrop:** reuse `.dialog-backdrop` (scrim `rgba(16,24,40,0.45)`, `z-index:100`). Backdrop click closes (mirrors Dialog's `onMouseDown` target check). Since search is read-only, backdrop-click-to-close is safe (no destructive guard needed).
- **Panel:** `max-width:640px`, full-width minus `--space-md` scrim padding, top-anchored at ~`15vh`. `background:--color-bg`, `border:1px solid --color-border`, `border-radius:--radius-md`, `box-shadow:--shadow-popover` (the elevated/floating elevation — heavier than `--shadow-card`, matching a command palette's "floats above everything" feel). Internal vertical layout: search-input row → results list → keyboard-hint footer.
- **Search-input row:** `padding:--space-md`, hairline `border-bottom:1px solid --color-border`. A muted `Search` icon (`--color-text-muted`, 16px) sits left of the input; the input itself is borderless inside this row (the row owns the chrome) but inherits the `.input` focus ring on the row when focused. Input is `--font-size-body`. **Autofocus on open.**
- **Results list:** scrollable region, `max-height:min(60vh,480px)`, internal overflow-y auto. The keyboard-active row auto-scrolls into view.
- **Keyboard-hint footer:** `padding:--space-sm --space-md`, `border-top:1px solid --color-border`, `--font-size-label`, `--color-text-muted`. Persistent across all states.

### Result-row anatomy (the load-bearing piece)
Each row is a `button`/`role="option"` reusing `.navrow` geometry (`min-height:--hit-min-height`, `padding:--space-md`, `gap:--space-sm`, `border-radius:--radius-sm`):

```
[type-icon] [Title (semibold, highlight=bold)         ] [type badge]
            [snippet … matched term in bold … (muted, label size, 1–2 lines, ellipsis)]
            [sub-line: "in {page title}" — muted, only for heading/attachment results]
```

- **Type icon** (16px, `--color-text-muted`): `FileText` for page, `Hash` for heading, `Paperclip` (or `AttachmentCard`'s `typeIconFor`) for attachment.
- **Title:** `--font-size-body` semibold; matched substrings bold (weight-only highlight). Single line, ellipsis-truncate (`.navrow .tree-label` pattern).
- **Type badge:** reuse `.role-badge` (surface fill, hairline border, `--radius-sm`, label type). Text: `Page` / `Heading` / `Attachment`. Right-aligned. Neutral — no accent.
- **Snippet:** `--font-size-label`, `--color-text-muted`, max 1–2 lines, `line-clamp`, matched terms bold. Source = Bleve fragment highlight, sanitized to weight-only markup.
- **States:**
  - hover → `--color-tree-hover` background.
  - keyboard-active (the single highlighted row) → `.navrow-active` treatment: `--color-tree-active-bg` + `inset 2px 0 0 --color-accent` + accent title. Exactly one row is active at a time.
  - hover and keyboard-active are visually distinct; mouse-move sets active to the hovered row (Obsidian behaviour).

### Result grouping
- Results are grouped by type under muted group labels (`Pages`, `Headings`, `Attachments`), `--font-size-label`, `--color-text-muted`, `padding:--space-sm --space-md`, optional `--color-surface` sticky header background. Empty groups are omitted entirely. Keyboard navigation flows across group boundaries as one continuous list (Enter opens the active row regardless of group).

### Top-bar trigger
- Mounts in `.topbar-right` (left of `UserMenu`, consistent with `repo-health`). A `.btn-ghost` button: `Search` icon + the word `Search` + a muted `⌘K` keycap. Min hit area `--hit-min-height`; icon-only fallback uses `--hit-min-icon`. Clicking opens the palette. Global `⌘K` / `Ctrl K` listener also opens it (and is the primary path).

## Interaction & Keyboard Contract

Keyboard-first (it is a palette). Inherits Dialog's focus-trap/restore contract.

| Key | Behaviour |
|-----|-----------|
| `⌘K` / `Ctrl K` | Open the palette from anywhere; focus the input. If already open, keep open (or toggle-close — implementer's choice, default: keep open + reselect input). |
| Typing | Debounced live search (~150–250ms). Loading indicator only after ~150ms in flight. |
| `↑` / `↓` | Move the single active row up/down across all groups; wraps optional; auto-scroll active row into view. |
| `Enter` (`↵`) | Open the active result and close the palette. Page → page route; Heading → page route + scroll to that section/anchor; Attachment → its **owning page** (SRCH-05). Navigation is in-app (react-router). |
| `Esc` | Close the palette; restore focus to the trigger (Dialog's `previouslyFocused` restore pattern). |
| Backdrop click | Close (read-only; safe). |
| Mouse hover | Sets the active row (syncs with keyboard selection). |
| Click a row | Same as `Enter` on that row. |

**Accessibility:** palette container `role="dialog"` `aria-modal="true"` `aria-label="Search"`; input `role="combobox"` `aria-expanded` `aria-controls={listid}`; results list `role="listbox"`; rows `role="option"` with `aria-selected` on the active row; `aria-activedescendant` on the input points at the active row id. Focus trap + restore replicate `Dialog.tsx`. Respect `:focus-visible` accent ring. Empty/no-results/error panels are announced via `aria-live="polite"`.

## State Inventory (the 5 required states)

| State | Trigger | What renders |
|-------|---------|--------------|
| **Empty** | Palette open, query empty | Centered scaled-down empty-state: `Search your workspace` heading (`--font-size-heading`) + body (`--font-size-label`, muted) + persistent keyboard-hint footer. No spinner. |
| **Typing / loading** | Query in flight > ~150ms | `.spinner` + `Searching…` (muted, label) in the results region; previous results may stay dimmed underneath (no layout jump). |
| **Results** | ≥1 match | Grouped, typed result rows as specified; optional muted `{n} results` count; first row keyboard-active by default. |
| **No-results** | Query non-empty, 0 matches | `No matches` heading + `Nothing matched "{query}". Try a different word or check the spelling.` body (muted). No error styling. |
| **Error** | Search request fails | Inline `.banner-warning`-style row: `Search is unavailable` + `Something went wrong while searching. Try again in a moment.` Never shows server/index internals; `aria-live` polite. |

State transitions must not cause the palette panel to resize abruptly between query keystrokes (reserve the results region height; prefer dim-over-replace during loading).
