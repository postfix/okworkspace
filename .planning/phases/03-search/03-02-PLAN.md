## Phase Goal

**As a** signed-in teammate, **I want to** open a ⌘K search palette and find pages by title, body, or tag, **so that** I can jump to any page without leaving the keyboard.

<objective>
Deliver the user-facing half of the first vertical slice: the Obsidian-style ⌘K quick-switcher wired to the `GET /api/v1/search` endpoint built in 03-01. After this plan, a real user can press ⌘K, type, see typed PAGE results with weight-only highlighting, and open one in-app. This is the slice that turns "the backend can search" into "the user can search."

Scope of THIS plan: the palette overlay, result rows, zustand store, debounced react-query hook, the `search()` API client function, the top-bar trigger + global ⌘K listener, and all 5 UI states. It renders whatever typed kinds the endpoint returns; page rows are live now, heading/attachment rows render correctly the moment 03-03 starts returning those kinds (no further UI change needed).

Purpose: Make search usable by the team (ex-Obsidian users) with the exact focus-trap, keyboard, and weight-only-highlight contract from the UI-SPEC.
Output: `web/src/components/search/*`, the search store + hook, `client.search`, and the AppShell trigger.
</objective>

<interface_contract>
Consumes the 03-01 result DTO (a JSON array): each item `{ kind: "page"|"heading"|"attachment", title, path, snippet, anchor?, page_title? }`. Empty query → `[]`. This plan defines the matching TypeScript types in `client.ts`:

- `type SearchResultKind = "page" | "heading" | "attachment";`
- `interface SearchResult { kind, title, path, snippet, anchor?, page_title? }`
- `async function search(q: string): Promise<SearchResult[]>` — `GET /api/v1/search?q=...`, `credentials: "same-origin"`, returns `[]` for blank q without a network call, throws "Search is unavailable. Try again in a moment." on non-ok.

Navigation on open (react-router): page → `/app/page/${path}`; heading → `/app/page/${path}` then scroll to `anchor` (the anchor will resolve once 03-03 adds heading ids to the renderer); attachment → `/app/page/${path}` (owning page).
</interface_contract>

| T-03-09 | Information Disclosure | error state copy | mitigate | Error panel shows only "Search is unavailable / Something went wrong"; never server/index internals (UI-SPEC). |
