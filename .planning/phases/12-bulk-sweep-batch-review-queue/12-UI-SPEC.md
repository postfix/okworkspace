# Phase 12 ‚Äî UI Design Contract

> Visual and interaction contract for **Bulk Sweep & Batch Review Queue** (TAG-05, TAG-06): the admin **bulk-sweep start control** (untagged-by-default with an "all pages" toggle, a "started ‚Äî N pages queued" confirmation, plus in-progress / empty / error states) and the dedicated **tag-review route** (`/app/tag-review`) ‚Äî a backlog list of pages with pending suggestions, reviewed **one page at a time by REUSING the Phase-11 per-tag approval list**, with a batched **Apply approved** action, progress through the backlog, and the empty / loading / error / per-page stale states. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **FINAL phase of milestone v1.0 ‚Äî subsequent phase on a shipped app.** This contract LOCKS to the existing hand-rolled token system (`web/src/styles/tokens.css` + `web/src/styles/controls.css`). It introduces **NO new design language, NO new dependency, and NO new token.** It composes three already-shipped patterns:
> 1. the **Phase-11 `TagSuggestList` approval surface** (`web/src/components/TagSuggest.tsx`) ‚Äî REUSED verbatim per page in the queue (its per-tag rows, new-vs-existing badge, new-default-unchecked, select-all/clear, count line, and stale/empty/error states are inherited, **not re-specified here**);
> 2. the **admin "Rebuild graph index" section** in `Admin.tsx` (`.admin-section` / `.admin-section-heading` / `.admin-muted` / `.admin-section-row` + `.btn-secondary` + `"Starting‚Ä¶"` busy + muted `role="status"` confirmation + `.field-error role="alert"`) ‚Äî the analog for the **sweep-start control**;
> 3. the **Phase-10 lazy-route + nav-entry pattern** (`GraphView` at `/app/graph`, the `.navrow .navrail-trash-row` + `tree-icon` + `tree-label` + `navrail-row-active` + `aria-current` row, and the `isAdmin = role === "admin"` gate in `AppShell.tsx`) ‚Äî the precedent for the **`/app/tag-review` route + admin-visible nav entry**.
>
> **Scope guard:** This spec covers ONLY (a) the sweep-start control and (b) the review-queue route shell (backlog list + one-page-at-a-time review + batched apply + progress + states). It does **NOT** re-spec the per-tag approval row internals ‚Äî those are inherited from `11-UI-SPEC.md` / `TagSuggest.tsx`. Backend mechanics (`KindTagSuggest` job, the `tag_suggestions` staging table, endpoint shapes, the 409 floor, batch granularity) are owned by the phase plan; they appear here only where they drive a visible state.

**Rendering / state:** The sweep-start fires a `@tanstack/react-query` mutation (mirror `reindexGraphMut` in `Admin.tsx` ‚Äî async 202, muted confirmation, no blocking). The review route fetches the pending-queue list via react-query (mirror `useBacklinks` / the `/graph` query) and per-page suggestions on demand; the batched apply is a mutation. Per-page review **reuses the existing `TagSuggestList`** (its local checked-set + captured base-revision contract is unchanged). **No `dangerouslySetInnerHTML`** anywhere ‚Äî page titles, paths, and tag names render as React text children only (the locked stored-XSS guard, identical to `BacklinksPanel`'s `{b.title}` rule). The sweep control, the nav entry, and the review route are **admin-gated** (render only when `me.role === "admin"`, mirroring `AppShell.tsx`'s `isAdmin` and the `RequireRole(admin)` server subgroup ‚Äî the server gate is the real boundary; this is convenience).

**Mount points:**
- **Sweep-start control** mounts as a new `<section className="admin-section">` in `Admin.tsx`, directly below the existing "Search" and "Graph" rebuild sections (a peer "Tagging" / "Tag suggestions" section). It is admin-only by virtue of living on the `/admin` route.
- **Review route** is a new `React.lazy` route `/app/tag-review` wrapped in `AppShell` (exactly like `/app/graph` / `GraphView`), reachable from a new **admin-visible nav row** in `.navrail-body` (beside the existing Graph and Trash rows), shown only when `isAdmin`.

## Interaction Contract (maps to TAG-05, TAG-06)

| # | Requirement | Contract |
|---|-------------|----------|
| TAG-05 | Admin bulk sweep ‚Üí enqueues per-page suggestions ‚Üí produces a pending queue, writing nothing automatically | The **sweep-start control** lives in an admin-only `Admin.tsx` section. The **scope toggle defaults to "only untagged pages"** (the locked default); checking "include pages that already have tags" sweeps all pages. Clicking **`Suggest tags for pages`** fires an async start mutation (mirrors `reindexGraphMut` ‚Äî returns immediately, 202), shows `Starting‚Ä¶` while in flight, then a muted `role="status"` confirmation ‚Äî **`Started‚Ä¶ ‚Äî {n} pages queued for review.`** ‚Äî or **`Every page already has tags ‚Äî nothing to suggest.`** when the untagged scope finds no targets. The control surfaces **no** job/queue/worker internals (hidden-infra rule). The sweep **writes nothing** ‚Äî it only produces pending suggestions that appear on the review route as the background work completes. |
| TAG-06 (review) | A user reviews the queue and approves/rejects per page | The **`/app/tag-review`** route (admin-gated, reachable from the admin-visible nav entry) lists every page with pending suggestions as a backlog of rows (title + muted path + `{n} pending` chip). Selecting a row opens that page&rsquo;s suggestions **one page at a time** in the **reused `TagSuggestList`** surface ‚Äî per-tag checkbox rows, the `new` badge, **new tags default UNCHECKED**, select-all/clear, and a live `{n} selected` count (all inherited from Phase 11). **Rejecting** is per-tag (uncheck) or whole-page (`Skip for now`, which leaves the page in the backlog and writes nothing). A muted `aria-live` **progress line** (`{n} pages left to review`) tracks the backlog draining. |
| TAG-06 (apply) | Approvals route through the same byte-stable apply path, committed in batches | The per-page primary CTA is **`Apply approved`** ‚Äî the single accent action, **NEVER auto-focused** (the trust-gate inversion is inherited verbatim: `Skip for now` is DOM-first and the deliberate initial-focus target so a reflexive Enter cannot write; Esc + backdrop dismiss without applying). Apply sends the checked tags + the page&rsquo;s captured base revision through the **Phase-11 byte-stable apply path** (the server re-validates/normalizes and commits in **batches**, not one commit per page ‚Äî backend-owned). On success the page leaves the backlog and the progress line decrements. The UI never assumes its checked list is authoritative (server re-validates). |
| TAG-06 (resumable / stale) | The sweep is resumable and never bypasses approval; a per-page stale base-revision is surfaced | The review route reads from the **persisted pending queue**, so reloading the page / returning later shows the same backlog (resumability is visible: the queue survives navigation and worker restarts ‚Äî no client state required). The load-bearing safety guarantee is honored in the UI by having **no auto-apply affordance anywhere** ‚Äî every write is an explicit per-page `Apply approved`. When a page changed since its suggestion was staged (a **409 stale base-revision**), the reused surface switches to the inherited **stale state** (warning banner, Apply path removed, `Skip` / `Re-run`) ‚Äî that one page is handled individually without sinking the rest of the backlog, mirroring the per-page 409-floor-without-failing-the-batch contract. |

**Accessibility / motion:** The reused `TagSuggestList` brings its full focus trap + Esc + restore-focus-on-close + `aria-live` count (inherited ‚Äî not re-specified). Backlog rows are keyboard-navigable buttons/links with visible `:focus-visible` accent rings; the open row is marked selected (`aria-current` / `aria-selected` as appropriate). The progress line uses `aria-live="polite"`. Every interactive element (sweep button, scope toggle, nav entry, backlog row, footer buttons) meets `--hit-min-height` (40px) / `--hit-min-icon` (44px). The pending-count badge and "new" badge convey meaning with words + numbers, never color alone. Under `prefers-reduced-motion: reduce`, any spinner stops animating (inherited rule). No `dangerouslySetInnerHTML` on any title, path, or tag.
