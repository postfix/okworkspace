<objective>
Make Live mode actually render Markdown inline — the heart of EDIT-01 for the TEXT constructs. Build the live-preview `ViewPlugin` that walks the Lezer markdown syntax tree over the visible ranges and emits a `DecorationSet`: `Decoration.mark` for styling (bold/italic/heading/inline-code) and `Decoration.replace` to hide syntax markers (`**`, `*`, `#`, backticks, link brackets/URL), then drops the hide-decorations the selection overlaps so markers reveal for editing on the active line. Wire it into `liveExtensions` (06-01's seam) and flesh out the theme so the rendered output visually matches `MarkdownProse.css`.

After this plan, an editor in Live mode sees headings/bold/italic/links/inline-code/fenced-code render inline as they type, with Obsidian-style active-line marker reveal — and Source mode still shows raw text byte-identically. Image and GFM-table WIDGETS plus internal `.md` link click-navigation are the next slice (06-03).

Purpose: Deliver the felt "live preview" quality for text constructs on the working editor surface from 06-01.
Output: `livePreview.ts` ViewPlugin; expanded `theme.ts`/`LivePreviewEditor.css`; `liveExtensions` now wires the plugin; `livePreview.test.ts` green.
</objective>
