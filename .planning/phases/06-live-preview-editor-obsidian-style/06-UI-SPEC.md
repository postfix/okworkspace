# Phase 6 — UI Design Contract

> Visual and interaction contract for the Obsidian-style Live-Preview Editor. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Nature of this phase:** UI *replacement*, not greenfield. It swaps the `@uiw/react-md-editor` split-pane surface for a CodeMirror 6 live-preview surface and unifies read + edit rendering. The design tokens were locked in Phase 0 (`web/src/styles/tokens.css`); this phase introduces NO new tokens — it maps the existing token set onto a new CM6 rendering surface and adds the interaction contracts for marker reveal, the Live/Source toggle, and inline widgets.

## Design System

| Property | Value |
|----------|-------|
| Tool | none (token-driven per-component CSS by deliberate project decision — see CLAUDE.md "What NOT to Use" / Phase 0 tokens.css) |
| Preset | not applicable |
| Component library | none (raw `@codemirror/{state,view,commands,language}` + `@codemirror/lang-markdown` + `@lezer/markdown`; NO React wrapper — `@uiw/react-codemirror` rejected per CONTEXT) |

**Source of truth for tokens:** `web/src/styles/tokens.css`. The CM6 theme (an `EditorView.theme({...})` / `EditorView.baseTheme`) and the per-component `LivePreviewEditor.css` MUST reference these CSS variables only — never hard-coded hex/px (matches the existing convention in `MarkdownProse.css`, `controls.css`, `SearchPalette.css`). Dark theme is deferred but the tokens are already named semantically, so the CM6 theme must read variables (not literals) so a future dark theme works without touching the editor.

Declared values (must be multiples of 4) — inherited unchanged from Phase 0 `tokens.css`:

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px | inline code padding; gap between toggle icon and label; marker-reveal does not change layout |
| sm | 8px | gap in the Live/Source header toggle group; table cell vertical padding |
| md | 16px | code-block padding; table cell horizontal padding; gap between editor surface and frontmatter form; default paragraph rhythm |
| lg | 24px | h2 top margin; blockquote / hr vertical rhythm |
| xl | 32px | h1 top margin; major section break |
| 2xl | 48px | (unused this phase) |
| 3xl | 64px | (unused this phase) |

**Phase-specific dimension tokens (reuse existing, do NOT redeclare):**
- `--prose-max-width: 720px` — the live-preview content column width in BOTH read and edit mode (read mode currently uses it via `.markdown-prose`; the unified CM6 surface MUST match it so reading and editing are visually identical).
- `--editor-min-height: 480px` — minimum CM6 editor height (replaces the old `MDEditor height={480}`).
- `--tree-indent: 16px`, `--topbar-height: 56px`, `--navrail-width: 264px` — unchanged app chrome.

Exceptions: none. (Marker reveal MUST be layout-neutral: revealing `**`/`#`/link syntax on the active line must NOT shift surrounding lines vertically — use `Decoration.replace` widths/`Decoration.mark` that reserve space or reveal in-place. This is a hard interaction constraint, not a spacing exception.)

## Typography

Inherited unchanged from Phase 0 `tokens.css` — exactly 4 sizes, exactly 2 weights. The CM6 live-preview decorations MUST render to these exact values so the editor matches `MarkdownProse` byte-for-byte visually:

| Role | Size | Weight | Line Height | Applied to (CM6 decoration) |
|------|------|--------|-------------|------------------------------|
| Display | 28px (`--font-size-display`) | 600 (`--font-weight-semibold`) | 1.2 (`--line-height-display`) | `# ` H1 lines |
| Heading | 20px (`--font-size-heading`) | 600 (`--font-weight-semibold`) | 1.2 (`--line-height-heading`) | `## ` H2 lines |
| Body | 16px (`--font-size-body`) | 400 (`--font-weight-regular`) | 1.5 (`--line-height-body`) | paragraphs, list items, H3–H6 (semibold), table cells |
| Label | 14px (`--font-size-label`) | 400/600 | 1.4 (`--line-height-label`) | Live/Source toggle labels, table header cells, footer hints |

**Inline styling decorations (must match `MarkdownProse.css`):**
- Bold (`**x**` / `__x__`) → `--font-weight-semibold`, body size.
- Italic (`*x*` / `_x_`) → `font-style: italic`, regular weight.
- Inline code (`` `x` `` ) → `--font-family-mono`, `--color-code-bg` background, `--radius-sm`, `--space-xs` horizontal padding.
- Code block (fenced) → `--font-family-mono`, `--color-code-bg` block, `--radius-md`, `--space-md` padding (mirrors `.markdown-prose pre`).
- H3–H6 → body size, semibold (mirrors existing `.markdown-prose h3..h6`).

**Source mode:** the entire document renders in `--font-family-mono` at body size with NO decorations (raw Markdown, every marker visible) — this is the only mode where mono applies document-wide.

## Color

Inherited unchanged from Phase 0 `tokens.css` — 60 / 30 / 10 split (light theme; dark deferred):

| Role | Value | Usage in this phase |
|------|-------|---------------------|
| Dominant (60%) | `#ffffff` (`--color-bg`) | editor canvas background (both modes), page background |
| Secondary (30%) | `#f6f7f8` (`--color-surface`) / `#f2f3f5` (`--color-code-bg`) | active-line subtle highlight; code-span and code-block background; toggle hover |
| Accent (10%) | `#2563eb` (`--color-accent`) | see reserved-for list below |
| Destructive | `#dc2626` (`--color-destructive`) | NOT used in this phase (no destructive actions on the editor surface; delete/trash live in PageActionMenu, out of scope here) |

**Accent reserved for (explicit — never "all interactive elements"):**
1. Rendered Markdown links (`.prose-link` equivalent) — accent text, underline-on-hover (matches `MarkdownProse.css`).
2. The single primary CTA on the editor screen — **"Save page"** (`.btn-primary`).
3. Focus-visible ring on the CM6 surface, the Live/Source toggle, and the frontmatter inputs — 2px accent outline, 2px offset (matches the global `:focus-visible` rule).
4. The active/selected segment of the Live/Source toggle (accent text or 2px accent underline on the active mode label) — low-emphasis; do NOT fill the whole toggle with accent.

Borders/dividers use `--color-border` (`#e3e5e8`); muted helper text (footer hints, "Source"/"Live" inactive label) uses `--color-text-muted` (`#6b7280`). The active-line marker-reveal background, if any, uses `--color-surface` — never accent.

## Live-Preview Interaction Contract (phase-specific)

> These are the load-bearing interaction decisions the executor and ui-checker must hold the implementation to. All flow from CONTEXT.md locked decisions.

### Mode toggle (Live ⇄ Source)
- A header toggle with two labeled segments, **Live** (default) and **Source**, placed in the editor toolbar row alongside the existing `LinkPicker`.
- Default mode = **Live**. Last-used mode persisted in `localStorage` via a zustand store (follow the existing `web/src/stores/recent.ts` store pattern; persistence key e.g. `okf.editor.mode`).
- `Cmd/Ctrl-E` toggles the mode globally on the editor surface (Obsidian parity). Must not conflict with browser/native shortcuts in the editor focus context.
- One shared `EditorState`; the toggle swaps only a decoration `Compartment`. **No serialization/reparse on toggle** → byte-identical by construction (success criterion 2). The toggle MUST NOT touch the document, the autosave timer, or `baseRevision`.

### Marker reveal (the Obsidian "Live Preview" feel)
- In Live mode, syntax markers (`**`, `*`, `#`, `` ` ``, link `[]()` brackets/URL, list bullets where decorated) are **hidden** via `Decoration.replace`/`Decoration.mark` EXCEPT on the line containing the cursor or within the active selection, where the raw markers are **revealed** so the user can edit them.
- Reveal MUST be layout-neutral (no vertical reflow of neighboring lines when the cursor enters/leaves a line).
- Reveal is driven by selection/cursor position, recomputed on `ViewPlugin.update` — not on a timer.

### Inline render set (v1 — locked, do not defer)
Headings, bold, italic, lists, links, inline code, fenced code blocks, **inline images**, **GFM tables**.
- **Inline images** `![alt](src)` → real `<img>` **widget** decoration. `src` MUST be sanitized/validated before the widget mounts: allow only the app's attachment/relative paths and safe schemes (`http(s)`, app-relative); reject `javascript:`/data-exec vectors. Broken/blocked `src` falls back to showing the raw `![alt](src)` markdown text (never a broken-image chrome that implies the bytes changed). `alt` is set on the `<img>`.
- **GFM tables** → styled grid widget. Must visually agree with `MarkdownProse.css` table styling (`border-collapse`, `1px solid --color-border`, `--space-xs`/`--space-sm` cell padding) AND with the server's Goldmark GFM / read-mode remark-gfm output.

### Unified read + edit rendering
- Read mode (`PageView`) and edit mode share ONE live-preview rendering surface; read mode is a **read-only** CM6 live-preview (no cursor caret affordance for editing, but selection/copy still works).
- The unified renderer MUST preserve every guarantee `MarkdownProse` provides today (this is the quality bar — ui-checker validates against it):
  1. **Internal `.md` link navigation** within the SPA (D-06) via `resolveRelativeMdLink(currentPath, href)` → `navigate("/app/page/<target>")`; external/non-`.md` links render unchanged.
  2. **GitHub-style heading anchor `id`s** (github-slugger algorithm, matching `okf.ScanHeadings`) so search deep-links (SRCH-06 / T-03-15) land on the right heading. The id MUST equal the slug — never `user-content-`-prefixed.
  3. **No raw HTML from page content** — only controlled mark/widget decorations. Raw-HTML blocks in content stay styled-as-text, never executed DOM (preserves the Phase 1/2/3 stored-XSS guard; rehype-raw-OFF equivalent).
- Whether read mode reuses `LivePreviewEditor` read-only or a thin shared rendering module is Claude's discretion at plan time — bounded by the three guarantees above. `MarkdownProse`'s retire-vs-retain is resolved at plan time under the same constraints.

### Preserved editor machinery (do NOT touch)
- `runSaver`, `bodyRef`/`frontmatterRef`, `baseRevision`, the `saving` single-flight guard, `DRAFT_DEBOUNCE_MS` autosave, the 409 ConflictBanner, `AutosaveStatus`, and the frontmatter form stay exactly as in `PageEditor.tsx`.
- The new editor exposes the SAME contract as `<MDEditor>`: `value: string` + `onChange(value: string)`. The CM6 document IS the raw Markdown string (no block model).
- `LinkPicker` stays wired into edit mode (relative `.md` insert, D-05/D-06).
