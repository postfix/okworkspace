<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| image `src` / link `href` in page content → DOM | Untrusted attributes cross into mounted `<img>`/link DOM where a script scheme would be an XSS vector. |
| table/image cell text → widget DOM | Untrusted content is placed into widget elements. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-06 | Tampering/Elevation | `![alt](javascript:…)` image src | mitigate | ImageWidget.toDOM() calls `sanitizeImageSrc` (http(s)/app-relative only); blocked → raw-text span. Tested in widgets.test.ts with the explicit `javascript:`/exec `data:` cases. |
| T-06-07 | Tampering | `[text](javascript:…)` link href | mitigate | linkNav reuses `resolveRelativeMdLink`'s scheme allowlist; null (external/unsafe) → default, never navigate; the link-mark carries href as data, never an executed action. |
| T-06-08 | Tampering | widget DOM from untrusted cell/alt text | mitigate | All widget DOM built via createElement + textContent/explicit attributes; NEVER innerHTML of page content (Anti-Pattern). widgets.test.ts asserts on element structure, not HTML strings. |
| T-06-09 | Information Disclosure | `data:` image exfil/abuse | mitigate | sanitizeImageSrc rejects executable `data:` by default (CONTEXT lists data-exec vectors as out of scope). |
| T-06-SC | Tampering | npm installs | accept | No new packages this plan. |
