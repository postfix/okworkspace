## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| page content (Markdown bytes) → editor render surface | Page body is user/agent-authored, untrusted; it crosses into the CM6 view where it could carry XSS vectors (image src, raw HTML, link href). |
| npm registry → build | New CM6/Lezer packages enter the supply chain. |
| client onChange → existing save path | Edited bytes flow to the unchanged save machinery; must remain verbatim (no injection of a transformed/escaped form). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Tampering/Elevation | image `src` rendered into a widget | mitigate | `sanitizeImageSrc` allowlist shipped this plan (Task 2): http(s)/app-relative only; javascript:/vbscript:/exec data: → null. Image widgets in 06-03 MUST call it before mounting `<img>`. |
| T-06-02 | Tampering | raw HTML in page body | mitigate | CM6 renders controlled decorations only; this plan never sets innerHTML from page content. Verified by construction — the minimal Live surface emits no widgets yet, and the contract for 06-02/06-03 is createElement+textContent only. |
| T-06-03 | Tampering | byte integrity of saved content | mitigate | The CM6 doc IS the raw Markdown; updateListener ships `doc.toString()` verbatim; toggle dispatches carry `effects` only (no `changes`). mode.test.ts + LivePreviewEditor.test.tsx + the backend corpus gate assert this. |
| T-06-SC | Tampering | npm installs (6 CM6/Lezer packages) | mitigate | Task 1 blocking human-verify checkpoint; RESEARCH Package Legitimacy Audit (no postinstall scripts; SUS verdicts are documented false positives). |
</threat_model>
