## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| page content → read-only render surface | The read surface renders untrusted page bytes; the same XSS vectors (image src, raw HTML, link href) apply as in edit mode. |
| heading text → DOM id attribute | Heading text becomes a DOM id; must be slugged (alnum/-/_ only), not raw. |
| location.hash → scroll target | The #hash drives a scroll lookup; must not be used to inject markup. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-10 | Tampering | read surface rendering untrusted content | mitigate | Read mode reuses the SAME controlled-decoration pipeline (sanitizeImageSrc, no-innerHTML widgets, link scheme gate) built in 06-02/06-03 — no separate, weaker read renderer. |
| T-06-11 | Tampering | heading id attribute from heading text | mitigate | id is `slug(text)` (lowercase, alnum/-/_/space→-, all other runes dropped) — cannot contain quotes/markup; matches okf.ScanHeadings. headingAnchors.test.ts asserts the exact output. |
| T-06-12 | Tampering | location.hash → scroll lookup | mitigate | hash is used only to LOOK UP a matching heading-line id and scroll; it is never written into the DOM or used to build markup. |
