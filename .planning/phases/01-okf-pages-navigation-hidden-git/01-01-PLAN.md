<objective>
Build the load-bearing spine every later Phase-1 slice depends on: the byte-stable OKF document model (`internal/okf`) with its golden-corpus round-trip test as the phase exit gate, the surgical required-frontmatter repair (PAGE-09), the real `CommitJob` handler that replaces the Phase-0 no-op stub (so a page write becomes one hidden Git commit, VER-01 mechanism), and the autosave `drafts` migration that later slices read/write.

Purpose: Round-trip rot is THE phase risk (RESEARCH Pitfall 1). The defense is structural and must exist before any editor ships — body is opaque text, frontmatter is edited surgically via `yaml.Node`, and the golden corpus pins byte-stability. This plan stands up the locked save & commit model spine: autosave-draft + batched-commit (D-01), draft-in-SQLite with the canonical .md written only when the CommitJob fires (D-02), commit triggered on explicit Save or idle (D-03), and routed through the existing single-writer Git spine (D-04). Routing every mutation through one `CommitJob` on the single-writer worker is the structural defense against `index.lock` collisions (Pitfall 2) and keeps the "working tree == HEAD" invariant (D-02).
Output: `internal/okf` package (Parse/Repair/Emit + golden corpus), `internal/pages/commitjob.go` (the D-04 single-writer commit handler), `internal/store/migrations/0004_drafts.sql` (the D-02 draft store), and `cmd/okf-workspace/main.go` wiring.
</objective>

## Trust Boundaries

|----------|-------------|
| HTTP handler → repo filesystem | Page bytes destined for disk cross here; must funnel through repo.Resolve + the single-writer commit, never direct os.* / git |
| Job payload → git CLI | Serialized commit payloads are executed by the worker; paths must be resolver-validated before staging |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | CommitJob fileWrite.Path | mitigate | `repo.Write` routes every path through the fuzz-tested SEC-01 resolver; `gitstore.Commit` re-resolves each staged path as a backstop (commit.go:40). No `os.*` in the okf or commitjob packages. |
| T-01-02 | Tampering/EoP | git invocation from CommitJob | mitigate | All git runs go through `gitstore.git()` arg-slice exec (never a shell string); CommitJob calls `gitstore.Commit`, never `exec.Command("git", ...)` directly. |
| T-01-03 | Tampering | Round-trip corruption of unrelated bytes (round-trip rot) | mitigate | Body is opaque text (no AST round-trip); frontmatter re-marshaled only when FrontDirty; golden-corpus byte-equality test is the exit gate. |
| T-01-04 | Information Disclosure | drafts table holding content | accept | Drafts are operational autosave buffers, not canonical content; same SQLite app.db trust level as sessions; no secrets stored. Files-as-truth invariant keeps canonical content on disk only. |
| T-01-SC | Tampering | Go module installs (goldmark/yaml.v3 already vetted) | accept | No new npm installs in this plan; backend deps (yaml.v3 in go.mod, goldmark not pulled by okf) are CLAUDE.md-locked and proxy-verified in RESEARCH. Frontend package install gate lives in Plan 02. |
