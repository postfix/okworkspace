# Phase 1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope note:** Phase 1 delivers the **core wiki loop** — page Read mode, page Edit mode (Markdown-with-preview), the live left navigation tree, page lifecycle (create / rename / move / delete-to-trash / restore), page-to-page linking, and hidden-Git version history + restore. It styles the surfaces that replace the Phase-0 `PLACEHOLDER_TREE` and the `MainPanePlaceholder` empty state inside the existing AppShell. It does NOT introduce attachments (Phase 2), search (Phase 3), the agent / DiffReviewDialog (Phase 4), or presence / soft-lock / full conflict-resolution UX (Phase 5). The only conflict surface here is the optimistic-concurrency **409 floor**.
>
> **Inheritance (LOCKED):** This phase inherits the entire Phase-0 design system verbatim — `web/src/styles/tokens.css` and `web/src/styles/controls.css` are the source of truth. Spacing scale, typography (4 sizes / 2 weights), color (60/30/10), radius/elevation, and the `Button` / `Input` / `Field` / `Dialog` / `Banner` / `Table` primitives are NOT re-derived. New tokens are ADDED only where a genuinely new surface (editor pane, tree interaction states, rendered Markdown prose, history list) needs them, and each addition is called out explicitly in the *New tokens this phase* sections below.

## Design System

| Property | Value |
|----------|-------|
| Tool | none (manual token system — inherited from Phase 0) |
| Preset | not applicable |
| Component library | none — hand-built token-driven primitives (`Button`, `Input`, `Field`, `Dialog`, `Banner`, `Table`) reused from Phase 0; new Phase-1 components are hand-built on the same tokens |
| Icon library | `lucide-react` (already in AppShell; only icons actually used are imported) |
| Font | Inherited: system UI stack (`--font-family-ui`); `--font-family-mono` for code blocks, inline code, and page-path/ID display |

**Why no shadcn (inherited decision):** Phase 0 documented the deliberate decision not to adopt shadcn (it would pull Tailwind + Radix outside the locked stack and contradict the single-binary / minimal-deps constraint). Phase 1 inherits this unchanged. The new editor/render/diff libraries this phase adds (`@uiw/react-md-editor`, `react-markdown` + `remark-gfm` + `rehype-sanitize`, `react-diff-viewer-continued`) are CLAUDE.md-locked and are **not** component registries — they are themed against the existing CSS custom properties. Registry vetting gate: **not applicable** (no shadcn / third-party registry).

**Token delivery:** Continue the Phase-0 pattern — all new tokens are CSS custom properties on `:root` in `web/src/styles/tokens.css`; components reference `var(--…)` only, no hard-coded px/hex. The third-party editor/render libraries are themed by mapping their CSS variables / overrides onto the existing tokens (see *Third-party component theming* below).

## Spacing Scale

Inherited from Phase 0 verbatim (8-point scale, all multiples of 4). No values change.

| Token | Value | Usage (Phase-1 surfaces) |
|-------|-------|--------------------------|
| `--space-xs` | 4px | Tree expand-caret↔icon gap, icon↔label gap, autosave-status inline gap |
| `--space-sm` | 8px | Tree row padding, nested-tree indent step, frontmatter form row gap, history-row padding-y |
| `--space-md` | 16px | Editor toolbar↔surface gap, gap between frontmatter form fields, dialog padding, page-meta block gap |
| `--space-lg` | 24px | Read-mode prose column padding, editor pane padding, section padding |
| `--space-xl` | 32px | Vertical rhythm between page title block and prose body |
| `--space-2xl` | 48px | Empty-state vertical offset (empty folder, empty trash, no-history) |
| `--space-3xl` | 64px | Reserved (page-level spacing); unchanged from Phase 0 |

**Layout fixed dimensions — inherited:** `--topbar-height: 56px`, `--navrail-width: 264px`. `--login-card-width` is unused this phase.

**New tokens this phase (layout/measure — multiples of 4 except the documented exception):**

| Token | Value | Usage |
|-------|-------|-------|
| `--tree-indent` | 16px | Per-level indent of a nested tree row (multiple of 4) |
| `--prose-max-width` | 720px | Max line-measure of rendered read-mode Markdown for readability (reading-measure, not a spacing token — exempt from the 4-multiple rule, like `--login-card-width` in Phase 0) |
| `--editor-min-height` | 480px | Minimum height of the Markdown editor surface so it doesn't collapse on short pages |

**Exceptions:** Inherited interactive-target rule — every tree row, toolbar button, editor control, history row action, and dialog button keeps the Phase-0 **40px minimum hit height** (`--hit-min-height`), 44px for icon-only buttons (`--hit-min-icon`). This is the only allowed deviation from the 4-multiple rule and applies to target sizing, not layout gaps. `--prose-max-width` is a reading-measure exception (parity with Phase-0 `--login-card-width: 400px`).

## Typography

Inherited from Phase 0 verbatim — **exactly 4 sizes, exactly 2 weights**. No new sizes or weights are introduced. Rendered Markdown body content (headings inside prose) maps onto the existing four sizes; it does NOT add new ones.

| Role | Size | Weight | Line Height | Phase-1 usage |
|------|------|--------|-------------|---------------|
| Display | 28px (`--font-size-display`) | 600 | 1.2 | Page title in Read mode; "New page" / "Move" / history dialog titles use Heading, not Display |
| Heading | 20px (`--font-size-heading`) | 600 | 1.2 | Rendered prose `#`/`##` map here; dialog titles; History panel title; section labels in the frontmatter form |
| Body | 16px (`--font-size-body`) | 400 | 1.5 | Default prose body text, editor text, tree row labels, frontmatter inputs, button text, history-row primary text |
| Label / caption | 14px (`--font-size-label`) | 400 | 1.4 | Field labels (Title/Tags/Description), helper text, tag chips, history-row meta (who/when), autosave status, "no SHAs" version labels, breadcrumb path |

**Rendered-Markdown heading mapping (read mode) — no new tokens, reuse the four sizes:**

| Markdown | Renders as | Token |
|----------|-----------|-------|
| `#` (h1) | Display | `--font-size-display` / 600 / 1.2 |
| `##` (h2) | Heading | `--font-size-heading` / 600 / 1.2 |
| `###`–`######` (h3–h6) | Body, semibold | `--font-size-body` / 600 / 1.4 (weight-only change; no new size) |
| paragraph / list / table cell | Body | `--font-size-body` / 400 / 1.5 |
| inline code / code block | Body size, `--font-family-mono` | `--font-size-body` / 400 / 1.5, mono family |

- Weights remain **only** 400 and 600. h3–h6 differentiate by weight (600), never by introducing a new size.
- Code uses `--font-family-mono` (already declared in Phase 0) — a family switch, not a new size/weight.
- Tag chips and history meta may use `font-variant-numeric: tabular-nums` for date alignment (presentation-only).

## Color

Inherited from Phase 0 verbatim — 60/30/10 split, light-theme baseline, semantic token names. No hex values change. Phase 1 adds a small number of **derived/neutral** tokens for surfaces Phase 0 didn't have (code-block background, tree hover/active states, diff add/remove tints), each documented and kept inside the existing budget (status/neutral signals only — never new large accent surfaces).

| Role | Value | Phase-1 usage |
|------|-------|---------------|
| Dominant (60%) — `--color-bg` | `#FFFFFF` | Read-mode prose surface, editor input surface, main pane |
| Dominant text — `--color-text` | `#1A1A1A` | Prose body, editor text, tree labels, page title |
| Secondary (30%) — `--color-surface` | `#F6F7F8` | Nav tree background (inherited), editor toolbar strip, history panel, code-block background base, tree hover |
| Secondary border — `--color-border` | `#E3E5E8` | Prose `hr`/table borders, editor pane border, tree dividers, history-row separators, blockquote bar |
| Muted text — `--color-text-muted` | `#6B7280` | Helper text, history meta (who/when), breadcrumb path, autosave "Saved" idle text, empty-state body, tag-chip text |
| Accent (10%) — `--color-accent` | `#2563EB` | See reserved list below |
| Accent text-on-accent — `--color-accent-fg` | `#FFFFFF` | Text/icon on accent fills |
| Destructive — `--color-destructive` | `#DC2626` | See reserved list below |
| Success (status only) — `--color-success` | `#16A34A` | "Saved" success tick on commit confirmation; restore-succeeded toast dot |
| Warning (status only) — `--color-warning` | `#D97706` | Stale-draft / conflict (409) banner, Git-divergence banner (inherited), unsaved-changes notice |

**Accent (`#2563EB`) reserved exclusively for (Phase-1 surfaces):**
1. The single **primary button** per surface/dialog: editor **"Save page"**, "New page" modal **"Create page"**, Move dialog **"Move page"**, History panel **"Restore this version"**, restore-from-trash **"Restore page"**.
2. The **active / current-page** tree-row indicator (left accent bar + accent label) — NAV-04.
3. **Internal page links** in rendered Markdown (read mode) — the in-app navigable `.md` links (D-06) render in accent (text-link treatment).
4. Input **focus ring** (inherited `:focus-visible`), including the editor surface, tree rows, and frontmatter inputs.
5. The editor↔preview **active tab/mode** indicator (when the md-editor exposes Edit/Preview toggles).

Accent is **never** used for: every interactive element, all icons, code-block syntax, tag chips, tree folder icons, table fills, or borders.

**Destructive (`#DC2626`) reserved exclusively for (Phase-1 surfaces):**
1. The **"Delete page"** action (rendered restrained as `btn-ghost-destructive` in the page action menu) and its confirmation dialog's confirm button.
2. The destructive-confirmation dialog accent (title icon + confirm button) for delete-to-trash.

Restore, rename, and move are **not** destructive (non-destructive, recoverable, history-preserving) and use accent/secondary, never destructive color.

**New tokens this phase (derived neutrals — declared in `tokens.css`, inside the budget):**

| Token | Value | Usage | Budget note |
|-------|-------|-------|-------------|
| `--color-code-bg` | `#F2F3F5` | Inline-code + fenced-code-block background (slightly deeper than `--color-surface` for separation) | Neutral surface tint, not accent — does not consume the 10% accent budget |
| `--color-tree-hover` | `var(--color-surface)` | Tree row hover background (alias, not a new hue) | Reuses `--color-surface`; no new color |
| `--color-tree-active-bg` | `rgba(37,99,235,0.08)` | Active/current tree row background wash behind the accent indicator | Low-alpha accent wash for the single active row only — within the accent reserved-for list (item 2) |
| `--color-diff-add` | `rgba(22,163,74,0.12)` | History "view this version" / future diff added-line tint (success-derived) | Status-signal tint only (history surface), low alpha; never a button fill |
| `--color-diff-remove` | `rgba(220,38,38,0.10)` | History removed-line tint (destructive-derived) | Status-signal tint only; never a button fill |

**Contrast:** All new text/background pairs meet WCAG AA. Code on `--color-code-bg` (`#1A1A1A` on `#F2F3F5`) passes; accent links (`#2563EB`) on `#FFFFFF` pass; muted history meta (`#6B7280`) on `#FFFFFF` passes body AA. The active-tree-row accent label on the low-alpha wash keeps a >4.5:1 ratio (accent text effectively on white).

**Radius & elevation — inherited:** `--radius-sm` (inputs, buttons, tag chips, tree rows), `--radius-md` (editor pane, history panel, dialogs, code blocks), `--shadow-card` (dialogs, floating link picker), `--shadow-popover` (page action menu, link picker dropdown).

## Copywriting Contract

Plain, calm, non-technical voice (the audience includes non-technical teammates). Sentence case, no jargon, no exclamation marks. **Hidden-Git rule (LOCKED, load-bearing):** the UI must NEVER surface Git vocabulary — no "commit", "branch", "SHA", "HEAD", "repository", "push", "merge", "diff" (as a noun the user reads), "checkout", "ref", or file paths. The user sees "Save page", "version", "history", "restore", and human page titles only. Errors say what happened and what to do next.

| Element | Copy |
|---------|------|
| Primary CTA — Save page (editor) | **Save page** |
| Primary CTA — Create page (modal) | **Create page** |
| Primary CTA — Create folder | **Create folder** |
| Primary CTA — Move page | **Move page** |
| Primary CTA — Restore version (history) | **Restore this version** |
| Primary CTA — Restore from trash | **Restore page** |
| Secondary action — Edit a page (read mode) | **Edit page** |
| Secondary action — Insert page link (editor) | **Link to page** |
| Page action menu items (read mode) | "Edit", "Rename", "Move", "Version history", "Delete" |
| New-page modal title | "New page" · field label: "Page title" · helper: "We'll create it in {folderName}." (path/filename never shown) |
| New-folder modal title | "New folder" · field label: "Folder name" |
| Rename modal title | "Rename page" · field label: "New title" · helper: "Links to this page will keep working." |
| Move dialog title | "Move page" · body: "Choose where this page should live." · helper: "Links to this page will keep working." |
| Autosave status — actively typing | "Saving…" (muted, with inline spinner) |
| Autosave status — draft saved, not yet a version | "Draft saved" (muted) |
| Autosave status — version cut (after Save / idle commit) | "Saved" (with success tick) |
| Unsaved-changes notice (navigating away with a dirty draft) | "You have unsaved changes. Save before leaving?" · confirm: "Save and leave" · secondary: "Leave without saving" |
| Empty state — main pane, no page open | heading **"Pick a page to get started"** · body "Choose a page from the left, or create a new one." |
| Empty state — empty folder | heading **"This folder is empty"** · body "Create a page here to fill it." (button: "Create page") |
| Empty state — page has no body yet | "This page is empty. Select Edit to start writing." |
| Empty state — version history, single version | "This page has only its first version so far. Every time you save, a new version is added here." |
| Empty state — trash empty | heading **"Trash is empty"** · body "Pages you delete will appear here, and you can restore them anytime." |
| Recently visited — empty | "Pages you open will show up here for quick access." |
| Recently visited — section label | "Recent" |
| Version history — panel title | "Version history" |
| Version history — row format | "{action} by {displayName} · {relativeTime}" — e.g. "Edited by Sam · 2 hours ago" / "Created by Sam · yesterday" (NO SHA, NO commit hash, NO branch) |
| Version history — view a version | link/button: "View this version" |
| Restore confirmation | Title: "Restore this version?" · Body: "This brings the page back to how it looked on {date}. Your current version is kept in history, so nothing is lost." · Confirm: "Restore this version" · Cancel: "Keep current version" |
| Save success toast (after commit) | "Page saved." |
| Restore success toast | "Page restored." |
| Error state — save conflict (409, optimistic-concurrency floor) | "This page was changed somewhere else since you opened it. Reload to see the latest version before saving again." · action: "Reload page" (Phase 1 floor: reload-and-retry; full overwrite/merge/save-as-copy choices arrive in Phase 5) |
| Error state — network/server on save | "We couldn't save your page just now. Your changes are kept here — check your connection and try Save again." (draft is safe in the editor) |
| Error state — title required (create) | "Give your page a title to create it." |
| Error state — title/name collision (create/folder) | none surfaced — the system auto-suffixes silently (D-12); user sees the created page |
| Error state — page not found (stale link / deleted) | "This page no longer exists. It may have been moved or deleted." · action: "Back to your workspace" |
| Error state — permission denied (reader attempting edit) | "You can read this page but not edit it. Ask an admin for editor access." (RBAC: readers view, editors edit) |
| Error state — restore-target collision (trash) | "A page with that name already exists, so this one was restored as '{title} (restored)'." (auto-suffix per D-10; informational, not blocking) |

**Destructive actions in this phase (each requires an explicit confirmation dialog — never a one-click destructive action):**
1. **Delete page → trash** (read-mode page action menu). Confirmation dialog:
   - Title: "Delete '{title}'?"
   - Body: "It will move to Trash. You can restore it anytime — nothing is permanently removed."
   - Confirm button: "Delete" (destructive)
   - Cancel: "Keep page"
   - *(D-11 nice-to-have, not required: if the page has inbound links, the body MAY add "Other pages link to this one; those links will stop working." — the planner may include this, but it is not required for MVP.)*

Rename, move, and restore are recoverable/non-destructive and do NOT use a destructive confirmation — rename/move are inline modals with a primary (accent) confirm; restore uses the accent confirm above. There is **no permanent-delete and no folder-delete** in Phase 1 (D-09), so those destructive flows are out of scope.

## Visual Focal Point (per Phase-1 surface)

Each surface declares the single element that anchors attention (one focal point per screen; everything else recedes).

| Surface | Visual focal point |
|---------|-------------------|
| Page Read mode (`/app/page/:path`) | The **rendered Markdown prose** in the centered reading column (`--prose-max-width`) is the focal point, led by the **page title** (Display). The left tree and top bar recede (secondary surface); the **"Edit page"** action sits top-right of the page as the primary affordance. |
| Page Edit mode | The **editor surface** (`@uiw/react-md-editor`, body) is the focal point; the **frontmatter form** (title/tags/description) sits above it as a compact secondary block; the **"Save page"** primary CTA + autosave status anchor the action. Tree/top bar recede. |
| Left navigation tree | The **current/active page row** (accent indicator) is the focal point within the rail; expand/collapse carets and folder/file icons are secondary; "Recent" is a quiet supporting list below the tree. The tree itself is a secondary surface relative to the main pane. |
| Version history (panel/dialog) | The **version list** is the focal point; the most recent version sits at top; each **"Restore this version"** is the per-row action anchor (accent), shown on the selected/hovered row so the list stays calm. |
| New-page / New-folder / Rename / Move dialog | The **single input** (title/name) is the focal point; the **primary CTA** (Create/Move/Rename) is the action anchor; helper text is supporting. |
| Trash view | The **list of deleted pages** is the focal point; each row's **"Restore page"** is the action anchor; empty-trash state recedes to muted copy. |

**Routes styled this phase:** `/app` (AppShell with live tree replacing placeholder), `/app/page/:path` (Read mode), `/app/page/:path` Edit mode (same route, mode toggle in zustand per RESEARCH), trash view (entry point within AppShell). Diff-review page mode (§18.3) is **Phase 4** — not built here.

**Interaction contracts (inherited + new):**
- Every interactive element keeps the inherited **focus-visible accent ring** (2px / 2px offset) — tree rows, editor, toolbar buttons, history rows, dialog buttons all keyboard-operable. This inherited focus-visible ring contract stays intact for all icon-only controls below.
- **Icon-only controls require an `aria-label`:** every icon-only interactive element (no visible text label) MUST carry an `aria-label` matching the action it performs. Specifically: the tree expand/collapse caret uses `aria-label="Expand {folderName}"` when collapsed and `aria-label="Collapse {folderName}"` when expanded (`aria-expanded` reflects state); the `@uiw/react-md-editor` toolbar buttons must expose accessible names via the library's accessibility API (its `commands` accept a `buttonProps`/name so each icon button announces its action, e.g. "Bold", "Insert link"); the `PageActionMenu` trigger (kebab/overflow icon) uses `aria-label="Page actions"`; any icon-only close (×) on dialogs uses `aria-label="Close"`. Icon-only buttons keep the 44px hit target (`--hit-min-icon`) and the inherited focus-visible accent ring.
- Tree: folders expand/collapse on row click or caret (NAV-02); the current page row is highlighted with the accent indicator (NAV-04); rows are now **interactive** (the Phase-0 `navrow-disabled` muted/no-hover style is removed for live page rows).
- Editor autosave never blocks typing; "Save page" (or ~5–10s idle, D-03) cuts a hidden version. The user never sees a SHA/commit — only the autosave status words.
- Internal page links in read mode navigate **within the app** (D-06) — they do not open a raw file or leave the SPA.
- Dialogs trap focus, close on Esc and backdrop click — **except** the `DeleteConfirmDialog`, where backdrop click does NOT confirm (only the explicit "Delete" button does), consistent with the Phase-0 destructive-dialog rule.
- Loading buttons show an inline spinner and disable; no layout shift (inherited).
- Dirty-draft navigation guard prompts the unsaved-changes dialog before leaving an edited page.
- RBAC: readers see Read mode + tree but no Edit/Create/Rename/Move/Delete affordances (or they render disabled with the permission copy); editors see all (exercises Phase-0 `RequireRole`).

**Third-party component theming (no new design system — themed to tokens):**
- `@uiw/react-md-editor`: disable/override its bundled theme; map its surface to `--color-bg`, toolbar strip to `--color-surface`, borders to `--color-border`, text to `--font-family-ui` / `--font-size-body`, code to `--font-family-mono` / `--color-code-bg`, focus to the accent ring. Min height `--editor-min-height`.
- `react-markdown` output (`MarkdownProse`): styled via a scoped class mapping to the heading sizes, `--color-code-bg`, accent links, `--color-border` table/hr/blockquote. `rehype-raw` stays **OFF**; `rehype-sanitize` **ON** (XSS).
- `react-diff-viewer-continued` (optional this phase per RESEARCH — history shows full versions, not side-by-side diff in MVP): if used, map added/removed line tints to `--color-diff-add` / `--color-diff-remove`, base to `--color-bg`/`--color-surface`. Reused by Phase-4 DiffReviewDialog.
