<objective>
Deliver the first end-to-end user capability of the wiki loop as a vertical slice: a user creates a page (title-only modal → slugged filename, D-12), edits its title/tags/description/body with autosave drafts, Saves to cut a hidden Git commit (writing the `.md` via the Plan-01 CommitJob), and views it rendered as Markdown — all navigable from a live left tree that replaces the Phase-0 `PLACEHOLDER_TREE`, with folder create, current-page highlight, and client-side recent pages.

Purpose: This is the core "create → edit → save → read → navigate" loop the whole phase exists to deliver. It exercises RBAC for the first time (readers view, editors mutate) and scaffolds the optimistic-concurrency 409 floor. After this plan a real user can do the central thing the product promises; subsequent plans add lifecycle (rename/move/link), trash, and history on top of this surface.
Output: `internal/pages` service + tree + handlers, the editor RBAC subgroup on the router, and the React Read/Edit/Tree surfaces with the new frontend render/editor deps.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → page API | Untrusted title/body/frontmatter/path + base_revision cross here |
