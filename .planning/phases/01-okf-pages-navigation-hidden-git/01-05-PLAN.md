## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | path + version token | mitigate | Path re-resolved through `repo.Resolve`; the version token is server-issued and used only for `git show <token>:<path>` via arg-slice exec — never interpolated into a shell. |
| T-05-02 | Information Disclosure | SHA / Git internals leaking to the UI | mitigate | History responses expose only action/who/when + an opaque token; no SHA/branch field is serialized (VER-02 "no raw SHAs"); frontend grep gate enforces it. |
| T-05-03 | Elevation of Privilege | reader restoring a version | mitigate | `RequireRole(auth.RoleEditor)` on the restore route; history/view reads open to any session; authorization from session. |
| T-05-04 | Repudiation | restore | mitigate | `audit.Record(Action restore)` on success; restore is a forward commit so the prior state remains in history. |
| T-05-05 | Tampering (data loss) | push force / auto-merge on divergence | mitigate | Push is ff-aware: on non-ff rejection it sets diverged and returns nil (alert, never `--force`/auto-merge), Phase-0 D-12 parity; Health surfaces the banner. |
