<objective>
Add the trash slice so deletion is a recycle bin, not destruction: delete-to-trash moves the page into `.okf-workspace/trash/` as a real Git commit (D-08), recording the original path + who + when (D-10); restore moves it back to its original folder, auto-suffixing on a name collision so a live page is never clobbered. Folders are implicit and there is no permanent-delete in Phase 1 (D-09).

Purpose: Success criterion 2 includes delete-to-trash and restore. Making delete recoverable (a `git mv`, history continuous) upholds the "nothing is truly destroyed" promise and the files-as-truth principle. After this plan a user can safely remove and recover pages.
Output: `internal/pages/trash.go`, the `0005_trash.sql` migration, trash handlers + routes, and the React DeleteConfirmDialog + TrashView surfaces.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → delete/restore API | Untrusted page path / trash id cross here |
| Trash path generation | Generated trash paths must stay inside `.okf-workspace/trash/` (no escape) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | delete path / restore target | mitigate | Source path + generated trash path + restore target all re-resolved through `repo.Resolve`; restore target collision-suffixed, never overwriting a live page. |
| T-04-02 | Elevation of Privilege | reader deleting/restoring | mitigate | `RequireRole(auth.RoleEditor)` on DELETE + restore routes; authorization from session. |
| T-04-03 | Repudiation | delete/restore | mitigate | `audit.Record(Action trash/restore)` on success. |
| T-04-04 | Tampering (data loss) | restore clobbering a live page | mitigate | `repo.Exists` collision check + auto-suffix (D-10); delete is a recoverable git mv, never `git rm` (D-08). |
| T-04-05 | Tampering | git mv args | mitigate | Move staged through `gitstore` via the CommitJob; no handler-side git, no shell string. |
| T-04-SC | Tampering | dependencies | accept | No new package installs in this plan. |
</threat_model>
