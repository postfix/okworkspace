<objective>
Add the page-lifecycle slice that keeps the workspace's links honest: rename and move a page (PAGE-04/PAGE-05) with an eager repo-wide rewrite of every inbound link committed in the same commit as the move (D-07), plus page-to-page linking (PAGE-08) via a "link to page" picker that emits a relative `.md` path and in-app navigation on click (D-06). The link rewrite is the second genuinely-new, subtle piece of the phase (after the round-trip core) and must go through the round-trip-safe `okf` edit path so rewriting a link never corrupts unrelated bytes.

Purpose: Success criterion 2 requires links to stay valid after rename/move. Eager rewrite (the locked D-07 strategy) keeps on-disk links literally correct at the cost of one repo walk per rename — acceptable at ~5 users. After this plan a user can reorganize the wiki (rename/move) and cross-link pages without ever breaking a link.
Output: `internal/okf/links.go` (structural link find/rewrite), `internal/pages/rename.go`, the rename handler + route, and the React RenameModal/MoveDialog/LinkPicker/PageActionMenu surfaces.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → rename/move API | Untrusted new title / target folder / source path cross here |
| Link rewrite → other pages' bytes | A rename mutates files the user did not directly edit (injection into other files) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | rename target / move parent path | mitigate | Slugify + reject `..`/absolute/NUL; every source/target/link path re-resolved through `repo.Resolve`; the destination is resolved too (symlink-escape covered by the resolver's EvalSymlinks). |
| T-03-02 | Tampering | inbound-link rewrite corrupting other files | mitigate | Rewrite is structural (Markdown link destinations only, code spans skipped) and routed through the round-trip-safe okf Emit; golden corpus + TestRename_NoCorruption guard byte-stability. |
| T-03-03 | Elevation of Privilege | reader renaming/moving | mitigate | `RequireRole(auth.RoleEditor)` on the rename route; authorization from session. |
| T-03-04 | Repudiation | rename/move | mitigate | `audit.Record(Action rename/move)` on success. |
| T-03-05 | Tampering | git mv args | mitigate | Move staged through `gitstore` arg-slice exec via the CommitJob; no shell string, no handler-side git. |
| T-03-SC | Tampering | dependencies | accept | No new package installs in this plan (frontend deps installed in Plan 02; backend okf is in-repo). |
</threat_model>
