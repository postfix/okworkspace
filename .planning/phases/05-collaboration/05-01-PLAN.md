<objective>
Slice 1 of Phase 5 (Collaboration): build the soft-lock store that every later slice depends on, as a pure, network-free Go package with a fully deterministic unit suite.

This is the COLL-02 foundation. It introduces `internal/locks` (model + service + GC), adds the `SessionConnectionIDKey` session const, and wires the `lock_gc` job + a ctx-gated ticker into `main.go`. No HTTP, no SSE, no UI — those land in Slices 2/3. Keeping the store pure (inject a clock, all I/O through `repo`) is what makes the load-bearing TTL/expiry/GC tests run with zero network and zero flakiness.

Purpose: a server-authoritative, file-backed, path-safe lock store is the source of truth for both soft locks (Slice 2) and presence (Slice 3). Lock truth must survive a single-tab refresh and never escape `.okf-workspace/locks/`.
Output: `internal/locks/{lock.go,service.go,gc.go,service_test.go}`, a one-line `internal/auth/session.go` const, and `main.go` wiring + GC ticker.
</objective>
