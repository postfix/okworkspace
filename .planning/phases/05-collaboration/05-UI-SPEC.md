# Phase 5 ‚Äî UI Design Contract

> Visual and interaction contract for the **Collaboration** surfaces (COLL-01..COLL-04). Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Nature of this phase:** an *extension* of a mature, LOCKED Obsidian-style design system shipped through Phase 7 (+ the Phase 4 agent UI) ‚Äî NOT a new visual language. The design tokens were locked in Phase 0 (`web/src/styles/tokens.css`), the shared control primitives in `web/src/styles/controls.css`, the `.banner`/`.banner-warning` pattern in `AppShell.css`/`CenteredCard.css`, and the `DiffReviewDialog` trust-gate in Phase 4. This phase introduces **NO new tokens** and **NO new color/type/spacing system**. It maps the existing token set onto three surfaces ‚Äî **PresenceIndicator** (new), **SoftLockBanner** (new), and **DiffReviewDialog `conflict` mode** (an extension of the Phase 4 dialog) ‚Äî and adds their interaction contracts.
>
> The load-bearing UX is **safety encoded as visual hierarchy**: a soft lock is *recoverable* (warning, force-edit allowed) and must read as a warning; a conflict's **Overwrite** is the *data-losing* choice and must read as destructive AND must never be the default-focused button; **Save as copy** / **Manual merge** are the safe choices and own the default focus. The diff must read as a **real diff, never prose** (inherited Phase 4 trust contract).

## Hard Constraints (carried from Phase 0‚Äì7 + Phase 4)

- **Token-only styling:** every new component (`PresenceIndicator.css`, `SoftLockBanner.css`) and the `DiffReviewDialog` conflict additions reference `var(--‚Ä¶)` only ‚Äî never hard-coded hex/px. Matches `controls.css`, `MarkdownProse.css`, `DiffReviewDialog.css`, `AppShell.css`.
- **Reuse existing primitives:** the `.banner` / `.banner-warning` pattern (already used for the Phase 1 409 floor in `PageEditor.tsx` ~line 220), `.btn` / `.btn-ghost` / `.btn-secondary` / `.btn-ghost-destructive` / `.btn-destructive`, the `AutosaveStatus` live-status pattern (`aria-live="polite"` + `Loader2`/`.spinner`), the existing `DiffReviewDialog` shell (focus-trap, Esc/backdrop **cancel-only**, deliberate non-primary initial focus), and `lucide-react` icons.
- **Extend, do NOT fork, `DiffReviewDialog`:** the Phase 4 Approve/Reject mode is **unchanged**. The conflict mode is a new branch in the SAME component (a `mode?: "review" | "conflict"` prop or a discriminated `conflict` prop set), preserving its focus-trap, Esc/backdrop-cancel, real-diff-always-rendered, and non-default-focus-on-the-consequential-action guarantees.
- **Hidden-Git rule (absolute):** no Git vocabulary anywhere ‚Äî no "commit", "repo", "push", "SHA", "merge conflict", "HEAD", "branch", "version control", "Git". Convey concurrency/versioning in plain language ("someone else edited this page", "your version", "their version", "save a copy"), mirroring `AutosaveStatus`, the Phase 1 conflict banner, and the Phase 4 stale-revision copy.
- **Editor-gated:** presence, soft locks, and the conflict dialog only appear inside Edit mode (`PageEditor`), which is already editor/admin gated. Readers never reach these surfaces.
- **Light theme only** (tokens are theme-ready; dark deferred). All values reference variables so a future dark theme works untouched.
- **The load-bearing safety rule (ROADMAP/CONTEXT):** force-edit is NOT a save bypass ‚Äî the revision check still runs and a stale save still opens the conflict dialog. The UI must therefore present force-edit as "take over editing" and NEVER imply it makes your save safe from a later conflict.

**Accent reserved for (explicit ‚Äî never "all interactive elements"):**
1. The keyboard **`:focus-visible` ring** (global 2px accent outline, 2px offset) on every focusable control: "Force edit", the three conflict footer buttons, the existing "Reload page".
2. The optional faint **your-own-presence** marker (`--color-tree-active-bg`, the existing rgba-accent tint) ‚Äî never filled accent, never on another user's presence.

**Critical color-hierarchy rules (ui-checker validates):**
- **SoftLock = warning, NOT destructive.** The soft-lock banner uses `--color-warning` (recoverable; you can force-edit). It must be visually distinguishable from the conflict surface. Never red.
- **Conflict / Overwrite = destructive.** The conflict dialog signals data risk; **Overwrite** is the only `--color-destructive` action in the dialog. **Manual merge** and **Save as copy** are neutral (`.btn-secondary` / `.btn-ghost`) ‚Äî they are the SAFE choices and must read calmer than Overwrite.
- **Presence = muted, never accent and never warning.** Presence is ambient awareness, not an alert. The exception is the transient **"Reconnecting‚Ä¶"** SSE-drop state, which uses warning to signal "your awareness is degraded."
- The diff added/removed tints come from the `react-diff-viewer-continued` theme (derive from `--color-success`/`--color-destructive` at low saturation) ‚Äî diff semantics, not the app's 10% accent; inherited unchanged from Phase 4.

## Surface Layout & Placement (phase-specific)

All three surfaces mount inside `PageEditor` (`web/src/routes/PageEditor.tsx`), whose root `.pageeditor` is a `--prose-max-width` (720px) flex column: `[conflict/error banners] ‚Üí .pageeditor-frontmatter ‚Üí .pageeditor-toolbar ‚Üí .pageeditor-surface ‚Üí .pageeditor-actions`.

### PresenceIndicator ‚Äî inline in the editor toolbar
- Mounts in the existing **`.pageeditor-toolbar`** row (the row that holds `LinkPicker` + the Live/Source `.pageeditor-mode` segment). It sits **left of the flex spacer** (before `.pageeditor-mode`'s `margin-left:auto`), so presence reads at the start of the toolbar and the mode toggle stays right-aligned. No new layout token.
- A single quiet line: a muted icon (`Pencil` for one other editor, `Users` for multiple) + Label-size muted text. **Never shows your own session.** When nobody else is editing, the indicator renders an empty `aria-live` span (occupies no visual weight ‚Äî mirrors `AutosaveStatus` idle).
- It is the awareness twin of `AutosaveStatus` (which lives in `.pageeditor-actions`): same restraint, same `aria-live="polite"` pattern.

### SoftLockBanner ‚Äî top-of-editor banner (reuses `.banner`)
- Mounts at the **top of `.pageeditor`**, in the SAME slot as the existing 409 conflict banner and `saveError` banner (PageEditor lines ~220‚Äì237), so all editor-level notices stack consistently above the frontmatter form. Full editor-column width (`--prose-max-width`).
- Built on `.banner banner-warning` (warning border on `--color-bg`, NOT a filled block) with `role="status"` (not `alert` ‚Äî it is informative, the editor is still usable via force-edit) and an internal flex row: `Lock` icon (warning) ¬∑ lead text ¬∑ *(flex spacer)* ¬∑ **"Force edit"** button (`.btn-secondary`, right-aligned, like the existing "Reload page" placement).
- When **you** hold the lock (or after you force-edit), the banner either disappears (you own it cleanly) or, when you forced past someone, flips to a brief confirmation state (see States) before clearing.

### Read-only editor treatment (under another's soft lock)
- While another session holds the lock and you have NOT forced edit, the `LivePreviewEditor` renders **read-only** (it already supports a read-only surface ‚Äî `PageView` uses it). Visual treatment: a faint `--color-surface` wash / reduced text contrast over `.pageeditor-surface` + a muted Label hint, so it visibly reads "view-only until you take over." The cursor does not enter the document; the existing `.pageeditor-actions` "Save page" CTA is disabled while read-only.
- **Force edit** transition: clicking "Force edit" in the banner flips the surface from read-only ‚Üí editable (the wash lifts, the caret enters, "Save page" re-enables) and the banner updates to the ownership/forced state. This transition is instant under reduced motion (no fade).

### DiffReviewDialog ‚Äî conflict mode (modal, inherited shell)
- The existing center-screen modal over a backdrop (focus-trapped, Esc/backdrop cancel-only). Opened by a stale-save **409** from either an explicit Save click or autosave: the client fetches the current server version and opens the dialog with `oldText = server version`, `newText = my version`, `mode = "conflict"`.
- Layout is the inherited Phase 4 layout (title ‚Üí optional summary ‚Üí **real diff** ‚Üí footer); only the **footer** changes for conflict mode (3 buttons instead of Reject/Approve). The diff body, view toggle (Side by side / Inline), internal scroll, and `--font-family-mono` are all inherited unchanged.

### Responsive / narrow-width
- PresenceIndicator and SoftLockBanner live inside the 720px editor column and inherit its responsive behavior; the banner row may wrap "Force edit" below the text at very narrow widths using the existing `--space-sm` gap (no new token).
- The conflict dialog is viewport-bounded (`max-width: 920px`, `max-height` with internal diff scroll) at all widths ‚Äî inherited from `DiffReviewDialog.css`. At narrow widths the **3 footer buttons wrap** (Overwrite isolated, Manual merge + Save as copy grouped) using `--space-sm`; Overwrite stays positionally separated from the safe pair (footer never renders three same-weight adjacent buttons).

## PresenceIndicator States

The indicator is `aria-live="polite"` so a screen reader hears presence changes without focus stealing (mirrors `AutosaveStatus`). Status changes are coalesced (announce on settle, not per SSE frame).

| State | Visual | Copy (Label, muted) | Notes |
|-------|--------|---------------------|-------|
| **Nobody else editing** | empty `aria-live` span (no icon, no weight) | (none) | Resting state; never shows your own session |
| **One other editor** | `Pencil` (14, muted) + text | "{name} is editing" | Single quiet line |
| **Multiple editors** | `Users` (14, muted) + text | "{name} and {N} others are editing" | N = others minus the named one; never counts you |
| **Connecting (initial SSE)** | `Loader2` (`.spinner`, 14) + text | "Connecting‚Ä¶" | First presence-stream connect; brief |
| **Reconnecting (SSE dropped)** | `AlertTriangle` (14, `--color-warning`) + text | "Reconnecting‚Ä¶" | The connection IS the heartbeat; on drop the client reconnects. Warning-tinted = degraded awareness, NOT an error toast |
| **Disconnected (gave up / offline)** | `WifiOff` (14, muted) + text | "Presence unavailable" | Optional terminal state if reconnection fails; muted, non-blocking ‚Äî editing still works |

- **Reduced motion:** the `Loader2` connecting spinner falls back to the static "Connecting‚Ä¶" text under `@media (prefers-reduced-motion: reduce)` (the global `.spinner` keyframe is suppressed; the label remains). Any "new editor joined" pulse/highlight on the indicator is also disabled under reduced motion (becomes an instant state swap).
- **No avatar stack, no color-coded user dots** (MVP, CONTEXT deferred). Names render as plain muted text.

| **Another session holds the lock** | yes, warning | `Lock` (`--color-warning`) | **"{name} is editing this page."** (lead, semibold) + " Your changes won't be saved until you take over." (regular) | **"Force edit"** (`.btn-secondary`) | **read-only** (wash + disabled Save) |
| **Force-edit in progress** | yes, warning | `Loader2` (`.spinner`) | "Taking over editing‚Ä¶" | button ‚Üí disabled "Taking over‚Ä¶" | transitioning |

- **Reduced motion:** the force-edit "Taking over‚Ä¶" spinner and the read-only-wash lift are instant under `@media (prefers-reduced-motion: reduce)`.

## DiffReviewDialog ‚Äî Conflict Mode Trust Contract (load-bearing)

> Extends the Phase 4 `DiffReviewDialog` with a `conflict` mode. It must read as a **real diff, never prose** (inherited). It encodes the safety hierarchy: **Overwrite is the data-losing choice and is destructive-tinted AND never default-focused; the safe choices (Manual merge, Save as copy) own the default focus.**

**Trigger:** a stale-save **409** (`ErrStaleRevision`) from an explicit Save OR from autosave. The client fetches the current server version and opens the dialog with `mode="conflict"`, `oldText = current server version`, `newText = my version`.

**Layout (top ‚Üí bottom) ‚Äî inherits the Phase 4 shell:**
1. **Title** (`.dialog-title`, Heading 20/600) ‚Äî **"This page was changed somewhere else"** (an `AlertTriangle` in `--color-destructive` may precede it to mark the data-risk state ‚Äî distinct from the warning-tinted stale state).
2. **Summary line** (Body, muted) ‚Äî a single plain sentence: **"Someone saved a different version while you were editing. Compare your version with theirs, then choose what to keep."** Caption above the diff, never a replacement for it.
3. **The real diff** ‚Äî `react-diff-viewer-continued`, split (side-by-side) by default with the inherited inline toggle. **Left (old) = "Their version" (on the server); Right (new) = "Your version".** Mono font, added/removed tints from the diff theme, internal scroll. A muted Label caption clarifies the columns ("Left: the saved version ¬∑ Right: your unsaved version") so "old/new" is never ambiguous in a conflict context.
4. **Footer** ‚Äî **three** buttons (see trust rules), replacing the Approve/Reject pair.

**Trust rules (mandatory ‚Äî ui-checker validates):**
- **Never prose-only:** the diff is always rendered (inherited). If `oldText === newText` (the "conflict" resolved itself ‚Äî both saved the same bytes), show "These versions are identical ‚Äî your save will go through." and offer a single safe "Save" ‚Äî never fabricate a diff.
- **Initial focus is NEVER on Overwrite.** Inheriting the Phase 4 inversion: focus lands on the **safest control** ‚Äî **"Save as copy"** (or "Manual merge", planner's discretion between the two safe options) ‚Äî or the diff scroll region. A reflexive Enter on open must NEVER discard the server's change. Document this with a comment so a future refactor doesn't "fix" it.
- **The three choices are visually ranked by risk, not equal weight:**
  - **Overwrite** ‚Äî `.btn-ghost-destructive` (destructive text on neutral surface, restrained) by default, OR `.btn-destructive` (filled) if the planner wants it unmistakable. It is the ONLY destructive-colored control in the dialog. It discards the server change. **Positionally isolated** from the safe pair (footer is space-between / wrapping groups Overwrite away from the safe two ‚Äî never three same-weight adjacent buttons).
  - **Manual merge** ‚Äî `.btn-secondary` (neutral). Closes the dialog and opens **your version in the editor with their version visible for reference** (e.g. a read-only side reference pane, or their version shown above/in a collapsible ‚Äî exact merge layout is Claude's discretion at plan time, but their version MUST stay visible while you reconcile). After reconciling you do a normal Save (which itself revision-checks).
  - **Save as copy** ‚Äî `.btn-secondary` (neutral). Creates a new page ("{Original Title} (Copy)", auto-deduped via `pages.Service.Create`/`uniquePath`) with your content; **the original page is never modified** and the server version is preserved. This is the zero-data-loss escape hatch and is a natural default-focus candidate.
- **Backdrop / Esc = cancel** (inherited): dismissing the dialog applies **nothing** ‚Äî your unsaved edits remain in the editor, the server version is untouched, and the conflict can be re-resolved on the next save. Esc/backdrop must NOT silently overwrite or silently discard.
- **Busy state:** Overwrite and Save-as-copy enter the inherited `busy` state on click (footer disabled, the chosen button shows "Saving‚Ä¶" / "Saving copy‚Ä¶"); Manual merge transitions immediately (no round-trip). On Overwrite, the save runs against the **current** revision ‚Äî and if yet another commit landed in the meantime, it 409s again and the dialog re-opens with the fresh server version (never a silent clobber).
- **Color-is-never-the-sole-signal:** Overwrite carries an explicit warning sub-line under the footer ("This replaces the other person's changes." ‚Äî Label, muted/`--color-destructive`) so its risk is stated in words, not just color; the diff `+`/`‚àí` gutters and line numbers carry change semantics (inherited).

**Destructive / consequential actions in this phase:**
- **Overwrite (conflict resolution)** is the one data-losing action ‚Äî it discards another person's saved changes. The safe alternatives (Save as copy = zero loss; Manual merge = reconcile) are always one click away and own the default focus.

## Accessibility

- **PresenceIndicator:** `aria-live="polite"` region (mirrors `AutosaveStatus`); presence changes announced on settle, coalesced (never per SSE frame). Container `aria-label="Who else is editing"`. Icons `aria-hidden="true"`; meaning carried by text. It is non-interactive (status only) and is skipped in tab order.
- **SoftLockBanner:** `role="status"` + `aria-live="polite"` (informative, not an `alert` ‚Äî the editor stays usable). The `Lock`/`AlertTriangle` icon pairs with lead text so warning color is never the sole signal. "Force edit" is a real `<button>` with a descriptive `aria-label`; it is in the normal tab order at the top of the editor.
- **Read-only editor state:** when under another's lock the editor surface is genuinely read-only (no caret, not just visually dimmed) and the "Save page" CTA is `disabled`; the muted hint explains why and how to take over. Screen-reader users hear the read-only state via the banner's live region, not a silent visual wash.
- **Conflict DiffReviewDialog:** inherits the Phase 4 focus trap + Esc + focus-restore-on-close. **Initial focus lands on a SAFE option (Save as copy / Manual merge) or the diff scroll region ‚Äî NEVER Overwrite** ‚Äî so the data-losing action cannot be triggered by a reflexive Enter. The three footer buttons are real `<button>`s with distinct accessible names; Overwrite is positionally and color-distinct. The diff scroll region is keyboard-scrollable and labeled (inherited). Tab order: diff region ‚Üí Save as copy ‚Üí Manual merge ‚Üí Overwrite (safe-first), with the wrap-trap (inherited) keeping focus inside the modal.
- **Reduced motion:** `@media (prefers-reduced-motion: reduce)` disables the presence connecting/reconnecting spinner glyph (falls back to static text), any "editor joined" pulse on the indicator, the force-edit "Taking over‚Ä¶" spinner, the read-only-wash fade, and any banner enter/exit transition (instant state swaps). The global `.spinner` keyframe suppression pattern (Phase 4) applies.
- **Color is never the sole signal:** soft-lock pairs warning color with the `Lock` icon + lead text; the conflict pairs destructive color with the `AlertTriangle` + title + the Overwrite risk sub-line; presence reconnecting pairs warning with the `AlertTriangle` + "Reconnecting‚Ä¶" text; the diff `+`/`‚àí` gutters + line numbers carry change semantics (inherited).
- Focus ring is the global 2px accent `:focus-visible` outline (2px offset) ‚Äî unchanged.
