<objective>
Slice 3 of Phase 5: deliver COLL-01 — per-page editing presence — by streaming a snapshot of current lock holders over SSE and rendering a quiet PresenceIndicator in the editor toolbar.

Presence is *derived from the lock store* (Slice 1): a "currently editing" user is a live lock holder. The backend clones the confirmed `handleExtractionStatus` SSE skeleton (Flusher + headers + ctx.Done cleanup + max-duration cap) and emits a full editors snapshot per tick from `lockStore.EditorsFor`. The frontend clones `subscribeExtractionStatus` into `subscribePresence` and renders the awareness twin of `AutosaveStatus`: muted, aria-live, never showing your own session.

Purpose: let a teammate see "Alice is editing" before colliding — ambient awareness, not an alert.
Output: handlers_presence.go + GET suffix dispatch + subscribePresence + PresenceIndicator + its toolbar mount.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → presence GET stream | A long-lived read stream; any authenticated user may observe (like reading a page), but it must not leak goroutines or echo unescaped names. |
| client conn query param → snapshot identity | Used only to mark you:true; never a path component. |
| username → rendered indicator text | Untrusted-ish display string crossing into the DOM. |

## STRIDE Threat Register (ASVS L1; block on high)

| Threat ID | Category | Component | Disposition | Mitigation Plan | Req |
|-----------|----------|-----------|-------------|-----------------|-----|
| T-05-11 | DoS | handlePresence SSE goroutine | mitigate | Clone handleExtractionStatus: `<-ctx.Done()` tears the goroutine down on client disconnect + an absolute sseMaxDuration cap reaps a forgotten tab. Verified by grep gates on ctx.Done + the cap const. | COLL-01 |
| T-05-12 | Information disclosure | presenceSnapshot | mitigate | The snapshot surfaces only username + you bool — never session ids, user ids, or another user's connection id. | COLL-01 |
| T-05-13 | Tampering (XSS) | username in PresenceIndicator | mitigate | Names render as React text (auto-escaped); no dangerouslySetInnerHTML; usernames constrained at creation. | COLL-01 |
| T-05-14 | Spoofing | conn query param | accept | The conn id is opaque and only marks you:true / dedups; it grants no privilege and is never a path component. A spoofed conn at worst mis-marks your-own-presence for that caller — no data risk. | COLL-01 |
