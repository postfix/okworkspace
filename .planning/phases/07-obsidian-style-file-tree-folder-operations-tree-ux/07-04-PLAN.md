<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| drag payload (dataTransfer) → drop handler | The dragged path is client-set; the server re-validates on the move call. |
| folder name / count → dialog + tree + trash row | Folder names are user content rendered as text. |
| grouped-restore button → API | The group id comes from the trash listing (server-issued). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-11 | Tampering | malicious/invalid drag path causing a self/descendant move | mitigate | dropAllowed prefix-guard rejects self/descendant/same-parent client-side; the server relocateFolder is the authoritative defense (07-01) and the collision/validation re-runs there. |
| T-07-12 | Tampering (stored XSS) | folder name in DeleteFolderDialog / grouped trash label | mitigate | Names render as React text nodes; no dangerouslySetInnerHTML; no raw HTML from folder names (UI-SPEC). |
| T-07-13 | EoP | reader triggers folder DnD/menu | mitigate | canEdit gates draggable + mutate menu items client-side; the server editor gate (07-01/07-02) is authoritative. |
| T-07-14 | Repudiation | folder op not attributable | mitigate | Server audits ActionFolderRename/Move/Trash/Restore (07-01/07-02); the client surface adds no new audit gap. |
| T-07-SC | Tampering | npm installs | accept | Zero packages installed; native HTML5 DnD + existing TanStack Query/lucide-react only (UI-SPEC forbids a DnD library). |
</threat_model>
