<objective>
Add grouped folder delete-to-trash and grouped restore on top of the existing per-page trash machinery. A new nullable `delete_group_id` column (migration 0008) tags every trash row produced by one folder-delete operation; `DeleteFolder` loops the existing per-page `Delete` over the enumerated descendant set under one shared group id; `RestoreGroup` iterates the existing per-page `Restore` over the group (index.md first so the folder exists), reusing the existing collision auto-suffix per page. Per-page delete/restore stay byte-identical (group id NULL). Folder delete/restore is per-page-looped under a shared group id — NOT strictly atomic across commits — exactly matching the existing `ReconcileTrash` (WR-01) stance; a `TestDeleteFolder_PartialProgress` test asserts graceful partial-progress behavior (the RESOLVED atomicity decision).

Purpose: Recoverable folder delete (TREE-04) and grouped folder restore / undo-folder-delete (TREE-05) — the backend behind the frontend DeleteFolderDialog + TrashView grouped row (Plan 04).
Output: `0008_trash_group.sql`; `DeleteFolder`/`RestoreGroup`/`deleteWithGroup` + grouped `ListTrash` in trash.go; grouped-restore HTTP route `/trash/group/{id}/restore` + folder-delete dispatch + `delete_group_id` in the trash listing response; `deleteFolder`/`restoreFolderGroup` API client functions + `delete_group_id` on the `TrashEntry` type; Go tests.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → HTTP handler | Folder `dir` and the grouped-restore `{id}` are attacker-controllable. |
| handler → pages service | Group id crosses into a SQL WHERE clause; dir crosses into file enumeration. |
| pages service → SQLite/git | delete_group_id is bound into INSERT/SELECT; each trashed page is committed individually. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-05 | Tampering / SQLi | `delete_group_id` in INSERT/SELECT | mitigate | Parameterized `?` binds only (deleteWithGroup INSERT, RestoreGroup SELECT) — never string-interpolated, matching existing trash code. `[VERIFIED: trash.go binds]` |
| T-07-06 | EoP | reader deletes/restores a folder | mitigate | Folder-delete dispatch + `/trash/group/{id}/restore` live in the `editor` subgroup `auth.RequireRole(auth.RoleEditor)` from the SESSION role. |
| T-07-07 | Tampering / EoP | traversal in folder `dir` for delete | mitigate | cleanPathString on the stripped dir at the handler edge + repo.Resolve in the worker; descendantPages only walks within the repo root and skips .git/.okf-workspace. |
| T-07-08 | DoS | restoring a huge group | accept | 5-user internal wiki; group sizes are small; per-page restore loop is bounded by the trashed set. No rate limit added (consistent with existing trash ops). |
| T-07-SC | Tampering | npm/go installs | accept | Zero packages installed (RESEARCH Package Legitimacy: not applicable). crypto/rand for the group id is stdlib. |
