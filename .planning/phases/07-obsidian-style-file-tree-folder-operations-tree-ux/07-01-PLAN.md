<objective>
Lift the single-page `relocate` (rename.go) to a folder-batch `relocateFolder` so a folder rename or move relocates the folder's `index.md` plus every `dir/` descendant `.md` AND rewrites every inbound link to every moved page in ONE commit, with the okf byte-stable round-trip preserved by construction. A folder move/rename into an already-existing target directory REJECTS with a clean 409 via a new `ErrFolderExists` sentinel (never a silent merge or auto-suffix). This is the byte-critical atomic path and ships first.

Purpose: Backend folder operations as a unit (TREE-02) with clean collision rejection (TREE-06 server side) — the foundation every frontend folder op (Plans 03/04) and the grouped-trash plan (Plan 02) builds on.
Output: `relocateFolder`/`RenameFolder`/`MoveFolder`/`descendantPages`/`ErrFolderExists` in `rename.go`; suffix-dispatched folder rename/move HTTP routes; `renameFolder`/`moveFolder` API client functions; Go tests covering the atomic commit, collision reject, and no-corruption cross-linked case.
</objective>
