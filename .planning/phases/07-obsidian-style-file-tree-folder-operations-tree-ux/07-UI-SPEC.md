# Phase 7 — UI Design Contract

> Visual and interaction contract for the Obsidian-style File Tree (folder operations & tree UX). Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Nature of this phase:** UI *clean rebuild*, not greenfield, and NOT a fresh visual language. The left file tree (`LeftTree`), its context menu (`TreeContextMenu`), and the shared `RenameModal` / `MoveDialog` are rewritten from scratch (deliberate CONTEXT override of the ROADMAP "do NOT re-do" note) for a consistent, maintainable base — but the **visual and interaction contract here is the existing one, preserved**. The design tokens were locked in Phase 0 (`web/src/styles/tokens.css`); this phase introduces **NO new tokens**. Every shipped behavior (page right-click menu, page drag-drop, folder-scoped create, dialog footers, the commit-wait tree update) must come through the rebuild with **no visual regression**, then the net-new folder operations, folder drag-drop, optimistic updates, and grouped trash restore layer onto that clean base. The felt-quality bar is "the tree feels like Obsidian's file explorer."

## Spacing Scale

Declared values (must be multiples of 4) — inherited unchanged from Phase 0 `tokens.css`. This phase adds no new spacing values:

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px (`--space-xs`) | gap between row icon/caret and label; gap between menu items; gap between restore-button icon and label; menu container padding |
| sm | 8px (`--space-sm`) | tree-row left padding base (`calc(depth * --tree-indent + --space-sm)`); context-menu item vertical padding; dialog-footer button gap; Trash row title/meta gap source |
| md | 16px (`--space-md`) | context-menu item horizontal padding; dialog-body gap; Trash row padding; dialog backdrop padding; the `--tree-indent` per-depth step |
| lg | 24px (`--space-lg`) | dialog inner padding (`.dialog`) |
| xl | 32px (`--space-xl`) | root drop-zone `min-height` (`--space-xl`); Trash view top padding |
| 2xl | 48px (`--space-2xl`) | (unused this phase) |
| 3xl | 64px (`--space-3xl`) | Trash empty-state vertical padding |

**Phase-specific dimension tokens (reuse existing, do NOT redeclare):**
- `--tree-indent: 16px` — per-depth indent step; nested rows indent by `calc(depth * var(--tree-indent) + var(--space-sm))` (the exact existing formula — keep it).
- `--navrail-width: 264px`, `--topbar-height: 56px` — unchanged app chrome.
- `--hit-min-height: 40px` — minimum height of every context-menu item and dialog footer button (existing).
- `--hit-min-icon: 44px` — minimum hit size for icon-only affordances (the expand/collapse caret); the caret may render a 16px glyph but its hit target should not regress below comfortable touch size.

**Exceptions:** none. (Context-menu container `min-width: 180px` and the `4px` viewport-clamp margin in `TreeContextMenu` are existing layout dimensions, not spacing-token exceptions — carry them unchanged.)

## Typography

Inherited unchanged from Phase 0 `tokens.css` — exactly 4 sizes, exactly 2 weights. The rebuilt tree, menu, and dialogs MUST render to these exact values (matches the existing components byte-for-byte visually):

| Role | Size | Weight | Line Height | Applied to (this phase) |
|------|------|--------|-------------|--------------------------|
| Display | 28px (`--font-size-display`) | 600 (`--font-weight-semibold`) | 1.2 (`--line-height-display`) | (unused on this surface) |
| Heading | 20px (`--font-size-heading`) | 600 (`--font-weight-semibold`) | 1.2 (`--line-height-heading`) | dialog titles (`.dialog-title`); Trash view title |
| Body | 16px (`--font-size-body`) | 400 (`--font-weight-regular`) | 1.5 (`--line-height-body`) | tree row labels; context-menu item labels; dialog body text; Trash row title (semibold variant); select/input controls in dialogs |
| Label | 14px (`--font-size-label`) | 400 (`--font-weight-regular`) | 1.4 (`--line-height-label`) | `lefttree-status` ("Loading…", error); Trash row meta ("Deleted by … · 2 hours ago"); `field-help` hints in dialogs; menu section affordance text |

**Weight discipline:** Regular (400) is default for all tree rows and menu items. Semibold (600) is reserved for: dialog titles, Trash row titles, and the active page label is **not** bolded (active state is shown by color + inset accent bar, not weight — keep the existing treatment). Do not introduce a third weight.

## Color

Inherited unchanged from Phase 0 `tokens.css` — 60 / 30 / 10 split (light theme; dark deferred):

| Role | Value | Usage in this phase |
|------|-------|---------------------|
| Dominant (60%) | `#ffffff` (`--color-bg`) | nav-rail / tree background; context-menu background; dialog panel background; Trash row background |
| Secondary (30%) | `#f6f7f8` (`--color-surface`) | row hover (`--color-tree-hover` = `--color-surface`); context-menu item hover/focus; Trash notice background |
| Accent (10%) | `#2563eb` (`--color-accent`) | see reserved-for list below |
| Destructive | `#dc2626` (`--color-destructive`) | the **Delete** context-menu item label (`.treemenu-item-danger`); the destructive confirm button in the folder/page Delete dialogs (`.btn-destructive`). Reserved for delete only. |

**Accent reserved for (explicit — never "all interactive elements"):**
1. **Active page row** — `--color-tree-active-bg` (`rgba(37,99,235,0.08)`) fill + a 2px inset accent left-bar (`inset 2px 0 0 --color-accent`) + accent label text. (Existing treatment — preserve exactly.)
2. **Drop-target highlight** on drag-over — a folder row or the root drop zone gains `--color-tree-active-bg` fill + a full 1px inset accent ring (`inset 0 0 0 1px --color-accent`) via the shared `.lefttree-droptarget` class. This is the single drag-over affordance for BOTH page and folder drags.
3. **Primary CTA buttons** in the rebuilt dialogs — "Rename", "Move page" / "Move folder", "Create page", "Create folder", "Restore folder" (`.btn-primary`).
4. **Focus-visible ring** — 2px accent outline, 2px offset on every focusable tree row, caret button, context-menu item, dialog control, and Trash restore button (global `:focus-visible` rule). The context-menu item uses `--color-surface` background on `:focus-visible` (existing) AND inherits the global accent outline.

**Invalid-drop affordance does NOT use accent or destructive color.** An invalid drop (folder onto itself or a descendant, or a no-op same-parent drop) shows the native `cursor: not-allowed` and applies **no** drop-target highlight (the row stays in its resting state) — a quiet, non-colored "this won't do anything" signal. Do not flash red; destructive color is reserved for the explicit Delete action only.

Borders/dividers use `--color-border` (`#e3e5e8`); the row caret, file/folder icons, and muted helper text use `--color-text-muted` (`#6b7280`).

The ONLY net-new copy is the folder-operation menu items + dialogs, the folder-delete confirmation (with page count), the grouped-restore affordance, and the optimistic-rollback error. No Git vocabulary is ever surfaced.

## Tree & Folder-Operations Interaction Contract (phase-specific)

> Load-bearing interaction decisions the executor and ui-checker must hold the implementation to. All flow from CONTEXT.md locked decisions (incl. the three user overrides: grouped restore, optimistic updates, clean rebuild).

### Context menu (rebuilt `TreeContextMenu`, behavior preserved)
- Cursor-anchored popup at `(clientX, clientY)`, viewport-clamped with a 4px margin, `role="menu"` + `role="menuitem"` rows, `--shadow-popover` on `--color-bg` with a `--color-border` hairline and `--radius-md`.
- **Keyboard/focus (preserve exactly):** opens with focus on the first item; Arrow Up/Down move focus (wrapping); Home/End jump; Enter/Space select; Esc closes; Tab is trapped (wraps with arrow semantics); focus is restored to the previously-focused element on close. Closes on outside-click, scroll (capture phase — the nav rail scrolls), and resize.
- Folder target shows the 5 folder actions; page target shows the 4 page actions; root (blank nav space) shows the 2 create actions. Readers (non-editor role) get a read-only menu: pages show only `Version history`; folders and root show an empty/suppressed menu (mirror the existing RBAC gate — same policy the server enforces).
- The `Delete` item is the only `danger`-styled item (destructive color text); it never gets accent.

### Drag-and-drop (extend the native HTML5 pattern; do NOT add a DnD library)
- **Pages** stay draggable for editors via `dataTransfer` key `application/x-okf-page` (existing). **Folders** become draggable AND droppable via a new key `application/x-okf-folder` carrying the folder path. Both pages and folders are valid drop payloads onto a folder row or the root drop zone.
- **Drag handle / cursor:** the whole row is the drag surface (no separate handle, matching Obsidian); draggable rows use `cursor: pointer` at rest and the browser's native grabbing cursor while dragging; `effectAllowed = "move"`.
- **Drop-target highlight:** on a valid drag-over, the target folder row or root zone gets `.lefttree-droptarget` (accent ring + tint). Exactly one target highlights at a time; it clears on `dragleave` and on drop.
- **Invalid-drop guard (TREE-06):** a folder dropped onto itself or any of its own descendants, and any same-parent no-op, is rejected client-side — NO highlight, `cursor: not-allowed`, and the move is never dispatched. This is computed from the dragged folder path vs. the hovered target path (prefix check), so the affordance is correct *during* drag-over, not only on drop.
- **Optimistic updates (CONTEXT override, TanStack Query):** on a DnD move AND on folder rename/move/delete, `onMutate` snapshots the `["tree"]` cache, applies the moved/renamed/deleted node to its new place **immediately**, returns the snapshot; `onError` rolls the cache back to the snapshot and surfaces the rollback error copy; `onSettled` invalidates `["tree"]` (and `["page"]`, `["trash"]` as relevant) to reconcile against the authoritative server commit. The moved node must appear in its new location with no spinner and no manual refresh. The shipped commit-wait remains the correctness backstop / reconciliation step.

### Folder operations (net-new; backed by the single-commit relocate)
- Folder **rename** and **move** open the rebuilt shared `RenameModal` / `MoveDialog` (the accessible, keyboard-only alternative to DnD). These dialogs handle BOTH page and folder targets, parameterized by node kind — same `Dialog` shell, focus trap, Esc/backdrop cancel (never confirms), and `.dialog-footer` layout as today.
- Folder **delete** opens the confirmation dialog naming the affected page count `N` (copy above), trashes every contained page (incl. `index.md`) as one group, then navigates the user back to their workspace if the open page was inside the deleted folder.
- Collisions (target dir exists) surface the clean-reject copy as a non-fatal `role="alert"` field error inside the dialog — the dialog stays open so the user can pick another name/destination; the tree is never silently merged.

### Trash grouped restore (net-new; layered on existing per-page restore)
- The Trash list (`TrashView`) gains **grouped entries**: pages trashed together by one folder-delete render as a single row labeled `Folder '{folderName}' · {N} pages` with a primary `Restore folder` button (icon `Undo2`), visually distinct from per-page rows only by the label and button copy — same `.trashview-row` chrome, border, radius, and spacing. Per-page restore remains available for individually-trashed pages.
- Grouped restore recreates the folder structure (paths incl. `index.md`) in one action; per-page collisions reuse the existing `restoredAlternative` auto-suffix and surface the batched notice copy (above) via the existing `.trashview-notice` `role="status"` region.

### Preserved app chrome / machinery (do NOT touch)
- The nav rail width, top "New page" / "New folder" buttons (which drive root-scoped create via the `LeftTreeHandle` imperative ref), expand/collapse caret behavior (NAV-02), active-row highlight (NAV-04), and recent-pages list are unchanged.
- `["tree"]` / `["page"]` / `["trash"]` are the TanStack Query keys; ephemeral UI (open menu, open dialog, drop-active flags) stays component-local / zustand — not server cache.
