# Phase 4 ÔÇö UI Design Contract

> Visual and interaction contract for the **Eino Agent** surfaces (AGNT-01..AGNT-11). Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Nature of this phase:** an *extension* of a mature, LOCKED Obsidian-style design system shipped through Phase 7 ÔÇö NOT a new visual language. The design tokens were locked in Phase 0 (`web/src/styles/tokens.css`) and the shared control primitives in `web/src/styles/controls.css`; this phase introduces **NO new tokens** and **NO new color/type/spacing system**. It maps the existing token set onto three NEW agent surfaces ÔÇö **PromptBar**, **AgentPanel**, and **DiffReviewDialog** ÔÇö and adds their interaction contracts. The felt-quality bar is "the agent feels native to the Obsidian-style shell the team already uses," and the load-bearing UX is **trust**: the diff must read as a real diff (never prose), and Approve/Reject must be unmistakable and not pre-focused on Approve.
>
> The single net-new token proposed below (`--agentpanel-width`) is an *additive layout dimension* in the existing style of `--navrail-width` ÔÇö not a redesign.

---

## Hard Constraints (carried from Phase 0ÔÇô7)

- **Token-only styling:** every agent component (`PromptBar.css`, `AgentPanel.css`, `DiffReviewDialog.css`) references `var(--ÔÇª)` only ÔÇö never hard-coded hex/px except where a token already encodes it. Matches the convention across `controls.css`, `MarkdownProse.css`, `LeftTree.css`, `Dialog.css`.
- **Reuse existing primitives:** `Dialog` (focus-trapped, **backdrop-never-confirms**), `.btn`/`.btn-primary`/`.btn-secondary`/`.btn-destructive`/`.btn-ghost`, `.select`, `.input`, `.field`/`.field-error`/`.field-help`, the `AutosaveStatus` live-status pattern (`.autosave-status` + `aria-live="polite"` + `.spinner`/`Loader2`), and `lucide-react` icons. **`DiffReviewDialog` is built on the existing `Dialog` shell**, parameterized for diff review.
- **Hidden-Git rule (absolute):** no Git vocabulary anywhere in agent UI ÔÇö no "commit", "repo", "push", "SHA", "version control", "Git". Convey retention/versioning in plain language ("the change is saved", "you can restore earlier versions"), mirroring `AutosaveStatus` and the Phase 1/2/7 history UX.
- **Editor-gated mutations:** the *propose-patch* and *rewrite-selection* affordances (and therefore the DiffReviewDialog Approve path) render only when `meData?.role === "editor" || "admin"` ÔÇö the same gate the editor and tree menus already enforce. Readers may Ask/Summarize and read answers, but never reach an Approve button.
- **Agent-off / no-provider gate:** when `agent.enabled === false` OR the provider is unreachable, the PromptBar renders **disabled** with an inline explanation ÔÇö never a silent hang (AI-SPEC ┬º6 "fail closed").
- **Light theme only** (tokens are theme-ready; dark deferred). The CM-adjacent surfaces and the panel reference variables so a future dark theme works untouched.

Established 8-point scale (`tokens.css`) ÔÇö inherited unchanged. This phase adds **no new spacing values**.

|-------|-------|---------------------|
| `--space-xs` | 4px | gap between mode-selector icon and label; gap between context chip icon and text; gap inside the streaming status indicator; submit-button icon/label gap |
| `--space-sm` | 8px | gap between PromptBar controls (mode select ┬À context chip ┬À workspace toggle ┬À submit); AgentPanel header control gap; gap between answer paragraphs' inline meta; diff summary line gap |
| `--space-md` | 16px | PromptBar inner padding (vertical) and horizontal control rhythm; AgentPanel body padding; gap between answer and "Propose this as a patch" affordance; DiffReviewDialog summaryÔåÆdiff gap |
| `--space-lg` | 24px | AgentPanel section padding; DiffReviewDialog inner padding (inherited from `.dialog`); empty-state vertical padding |
| `--space-xl` | 32px | PromptBar horizontal page padding to align with `.mainpane` (`--space-lg`) + control inset; major break between answer stream and panel footer |
| `--space-2xl` | 48px | AgentPanel empty-state top inset |
| `--space-3xl` | 64px | (unused this phase) |

**Hit sizes:** every PromptBar control, panel header button, and dialog footer button uses `--hit-min-height` (40px). Icon-only buttons (panel collapse toggle, "propose as patch" icon affordance, dialog close X) use `--hit-min-icon` (44px) min touch target.

**Phase-specific dimension tokens:**
- **NET-NEW additive token (proposed):** `--agentpanel-width: 360px` ÔÇö the fixed width of the right-side AgentPanel, declared in `tokens.css` alongside `--navrail-width: 264px` in the existing "Layout fixed dimensions" block. This is an additive layout dimension in the established style, **not** a redesign. (Rationale: a chat/answer column reads comfortably wider than the 264px nav rail; 360px is a multiple of 8 and keeps the center editor's `--prose-max-width: 720px` usable on common laptop widths.)

**PromptBar height:** the bar is content-sized (single-row of `--hit-min-height` controls + `--space-md` vertical padding Ôëê 72px); it does **not** introduce a fixed token ÔÇö it sizes from its tokenized padding so it stays in scale.

Exceptions: none. (The diff column min-width / wrap behavior is governed by `react-diff-viewer-continued`'s own layout, themed but not re-spaced; this is a media/library dimension, not a spacing-token exception.)

## Typography

Inherited unchanged from Phase 0 `tokens.css` ÔÇö exactly 4 sizes, exactly 2 weights. The agent surfaces MUST render to these exact values so they match the rest of the shell byte-for-byte.

| Role | Size | Weight | Line Height | Applied to (this phase) |
|------|------|--------|-------------|--------------------------|
| Display | 28px (`--font-size-display`) | 600 (`--font-weight-semibold`) | 1.2 (`--line-height-display`) | (unused on these surfaces) |
| Heading | 20px (`--font-size-heading`) | 600 (`--font-weight-semibold`) | 1.2 (`--line-height-heading`) | AgentPanel header title ("Assistant"); DiffReviewDialog title (`.dialog-title`) |
| Body | 16px (`--font-size-body`) | 400 (`--font-weight-regular`) | 1.5 (`--line-height-body`) | streamed answer prose; PromptBar mode `<select>` and submit label; DiffReviewDialog summary text; diff line text (rendered in **mono** but at body size) |
| Label | 14px (`--font-size-label`) | 400 (`--font-weight-regular`) | 1.4 (`--line-height-label`) | context chip text ("This page" / "Selection" / attachment name / "Whole workspace"); streaming status ("ThinkingÔÇª", "StreamingÔÇª"); panel meta (mode ┬À scope line); empty-state hint; agent-off message; stale-revision warning body; "Reasoned over N pages" citation line |

**Weight discipline:** Regular (400) is default for answers, chips, status, and controls. Semibold (600) is reserved for: the AgentPanel header title, the DiffReviewDialog title, and rendered-Markdown bold within a streamed answer (mirrors `MarkdownProse` bold = semibold). Do not introduce a third weight. The active mode label in the selector is **not** bolded ÔÇö selection is shown by the native `<select>` value, not weight.

**Streamed answer rendering:** answers render through the **same sanitized read-only Markdown surface as `MarkdownProse`** (headings, bold/italic, lists, links, inline/fenced code, GFM tables ÔÇö no raw HTML, rehype-raw OFF / stored-XSS guard preserved). This keeps agent output visually identical to page content and inherits the Phase 6 unified render guarantees. Inline code and code blocks use `--font-family-mono` + `--color-code-bg` exactly as elsewhere.

## Surface Layout & Placement (phase-specific)

The existing shell (`AppShell.tsx` / `AppShell.css`) is a vertical flex column: `.topbar` (56px) on top, `.appshell-body` (`display:flex`, `flex:1`) below containing `.navrail` (264px, left) + `.mainpane` (`flex:1`, the editor/page). The agent surfaces slot in as follows **without disturbing existing chrome**:

### PromptBar ÔÇö persistent bottom bar
- Renders as a new **third row** of the `.appshell` flex column, **below `.appshell-body`** (`flex: 0 0 auto`), spanning the full width under both the nav rail and the main pane ÔÇö a persistent footer bar. Background `--color-surface`, 1px top border `--color-border` (the mirror of `.topbar`'s bottom border).
- Single horizontal row, left-to-right: **mode `<select>`** ┬À **context chip** (auto-set, read-only indicator) ┬À **"Whole workspace" toggle** ┬À *(flex spacer)* ┬À **streaming/in-flight status** ┬À **Submit** (`.btn-primary`, accent, with `SendHorizontal` icon).
- Horizontal padding aligns the controls with `.mainpane` content (`--space-lg` page inset + `--space-md` control rhythm).
- The PromptBar is **always present** for authenticated users (it is the agent entry point, SPEC ┬º6.4) but renders **disabled** in the agent-off/no-provider state (see states).

### AgentPanel ÔÇö right-side collapsible column (3-column layout)
- Adds a **third column to `.appshell-body`**, to the RIGHT of `.mainpane`: `flex: 0 0 var(--agentpanel-width)` (360px), background `--color-bg`, 1px **left** border `--color-border` (the symmetric mirror of `.navrail`'s right border). Layout becomes: nav rail (264px) ┬À editor (`flex:1`) ┬À agent panel (360px).
- **Collapsible:** when collapsed the column is removed from the flow (width 0 / not rendered) and the editor reclaims the space. A toggle (`PanelRightOpen`/`PanelRightClose`, icon-only ghost button) lives in the **`.topbar-right`** cluster (next to search/user menu) AND the panel header carries a collapse affordance. Collapsed/expanded state persists in a zustand store (follow `web/src/stores/editorMode.ts` pattern; key e.g. `okf.agent.panelOpen`).
- **Auto-open on first submit:** submitting a prompt while the panel is collapsed opens it (so the answer is visible) ÔÇö but the user may re-collapse at will; the auto-open does not fight a deliberate collapse on subsequent turns.
- Internal structure: **header** (title "Assistant" + current mode┬Àscope meta + collapse button) ÔåÆ **answer region** (scrollable, `aria-live`) ÔåÆ optional **action footer** ("Propose this as a patch" when relevant).

### DiffReviewDialog ÔÇö modal (on the existing Dialog)
- A modal built on the existing `Dialog` shell (focus-trapped, Esc/backdrop **cancel-only**), opened by: the "Propose this as a patch" affordance in the AgentPanel, the Propose-patch mode, or the Rewrite-selection flow. It is **center-screen over a backdrop**, independent of the 3-column layout.
- It is **REUSED in Phase 5** (conflict resolution) ÔÇö so its props are general: `{ title, oldText, newText, summary?, onApprove, onReject, stale?, busy? }`. It must not hard-code agent-specific copy beyond what callers pass in.

### Responsive / narrow-width collapse
- **ÔëÑ 1280px:** full 3-column layout (nav rail ┬À editor ┬À 360px panel) renders comfortably; editor keeps `--prose-max-width: 720px`.
- **< 1280px (and the panel is open):** the AgentPanel **overlays** the right edge as a floating panel (position over `.mainpane`, `--shadow-popover`, same 360px) rather than squeezing the editor below a readable width ÔÇö OR the editor's effective width drops but never below `--prose-max-width`; if it would, the panel auto-switches to overlay. (Overlay vs. squeeze threshold is Claude's discretion at plan time, bounded by: the editor prose column must never fall below `--prose-max-width` while the panel is docked.)
- **< 900px (e.g. narrow laptop / split screen):** the AgentPanel is **collapsed by default** and opens as a full-height overlay from the right when toggled; the nav rail behavior is unchanged (existing). The PromptBar remains a full-width bottom bar; its controls may wrap to a second row at very narrow widths (mode/context on row 1, workspace toggle + submit on row 2) ÔÇö wrapping uses the same `--space-sm` gap, no new tokens.
- The DiffReviewDialog is viewport-bounded (`max-width`, `max-height` with internal scroll on the diff) at all widths ÔÇö inherited from `Dialog` + an internal scroll region on the diff body.

The context chip reflects the current view, auto-set (CONTEXT ┬ºPrompt UX):

| Current view | Chip label | Chip icon (lucide, 14) | Scope sent |
|--------------|-----------|------------------------|------------|

## DiffReviewDialog ÔÇö Trust Contract (load-bearing)

> This is the explicit human gate against indirect prompt injection (AI-SPEC ┬º1 #1/#3, ┬º6). It must read as a **real diff, never a prose summary.** Built on the existing `Dialog` (focus trap, Esc/backdrop **cancel-only ÔÇö backdrop never approves**).

**Props (general, reused in Phase 5):** `{ title, oldText, newText, summary?, onApprove, onReject, stale?, busy? }`.

**Layout (top ÔåÆ bottom):**
1. **Title** (`.dialog-title`, Heading 20/600) ÔÇö e.g. "Review this change" (patch) / "Review the rewrite" (selection). Caller-supplied.
2. **Optional one-line summary** (Body, muted) ÔÇö a *single* plain sentence of what changed (e.g. "Updated the deploy steps in the Setup section"). It is a **caption above the diff, never a replacement for it** ÔÇö the diff is always rendered in full.
3. **The real diff** ÔÇö `react-diff-viewer-continued`, **split (side-by-side) view by default** with an inline-view toggle; old (left) vs new (right); **mono font** (`--font-family-mono`); added lines tinted from `--color-success` (low saturation), removed from `--color-destructive` (low saturation), unchanged on `--color-surface` gutter; line numbers shown. The diff body scrolls internally (viewport-bounded) so long patches stay reviewable.
4. **Footer** (`.dialog-footer`) ÔÇö **Reject** (left/secondary) and **Approve** (right/primary). See trust rules.

**Trust rules (mandatory ÔÇö ui-checker validates):**
- **Never prose-only:** the diff component is always rendered. If `oldText === newText` (no-op), show "No changes were proposed." and disable Approve ÔÇö never fabricate a diff.
- **Approve is primary (accent) but NOT auto-focused.** The Dialog's focus-into-modal lands on the **first focusable control = Reject** (or the diff scroll region), so the user cannot approve by reflexively hitting Enter on open. This is a deliberate inversion of the usual "primary is focused" pattern, because the consequential action is the write. Document this in the component (a comment) so a future refactor doesn't "fix" it.
- **Approve and Reject are unmistakable and distinct:** Approve = `.btn-primary` (accent) labeled **"Approve & save"**; Reject = `.btn-secondary` with destructive-tinted text (or `.btn-ghost-destructive`) labeled **"Reject"**. They are visually and positionally separated (footer space-between), never two same-weight buttons.
- **Backdrop / Esc = cancel = reject-without-applying** (inherited `Dialog` contract): dismissing the dialog discards the proposal; it never applies.
- **Busy state:** on Approve, the footer enters `busy` (buttons disabled, confirm shows "SavingÔÇª") via the existing `Dialog` busy pattern ÔÇö the apply round-trips to the gated `/apply-patch` endpoint.

**Stale-revision state (`stale === true`):** if the page moved between proposal and approval (AI-SPEC ┬º6 stale check ÔåÆ 409):
- A **warning banner** replaces the footer's Approve path: `AlertTriangle` (`--color-warning`) + heading "This page changed since the assistant read it." + body "Someone edited the page while you were reviewing. Re-run the assistant to get a fresh proposal." (Body = Label, muted.)
- **Approve is removed/disabled** in the stale state ÔÇö there is no path to apply a stale proposal. The footer offers **"Close"** (secondary) and optionally **"Re-run"** (primary, accent) which re-issues the proposal against the current revision. Warning color, never accent, marks the blocking condition.

## Accessibility

- **PromptBar keyboard nav:** native `<select>` for mode (arrow-key accessible by default); logical tab order (mode ÔåÆ workspace toggle ÔåÆ prompt input ÔåÆ submit); the read-only context chip is `aria-hidden` from tab order but exposed to SR via the panel meta line; Enter submits, Shift+Enter newlines, Esc cancels in-flight; a focus-prompt shortcut hinted via the muted `.keycap`.
- **Mode selector** is a real `<select>` (not a custom listbox) so it inherits full keyboard + SR semantics for free ÔÇö consistent with the project's "hand-rolled but native-first" stance (Phase 7 used native `<select>` for move destinations).
- **Streaming live region:** the AgentAnswer region is `aria-live="polite"` `aria-busy` (true while streaming). Token appends are coalesced to sentence/settle boundaries so SR users hear coherent chunks, not per-character noise. The "ThinkingÔÇª/StreamingÔÇª" status is also `aria-live="polite"` (mirrors `AutosaveStatus`).
- **Reduced motion:** `@media (prefers-reduced-motion: reduce)` disables the streaming caret pulse, animated auto-scroll, and the rotating spinner glyph (falls back to static status text). The panel open/collapse transition is instant (no slide) under reduced motion.
- **DiffReviewDialog focus management:** inherits the `Dialog` focus trap + Esc + focus-restore-on-close. **Critically, initial focus lands on Reject (or the diff scroll region), NOT Approve** ÔÇö so the consequential write cannot be triggered by a reflexive Enter. The diff scroll region is keyboard-scrollable and labeled. Approve/Reject are real `<button>`s with distinct accessible names ("Approve & save" / "Reject").
- **AgentPanel** is a labeled region (`aria-label="Assistant"`); the collapse toggle is a real button with a state-reflecting `aria-label`. When the panel opens as an overlay at narrow widths it traps focus only if modal; as a docked column it does not trap (it is a peer landmark).
- All icon-only buttons (collapse, submit when icon-only, propose, dialog close) carry an `aria-label`; all lucide icons are `aria-hidden="true"`.
- Focus ring is the global 2px accent `:focus-visible` outline (2px offset) ÔÇö unchanged.
- Color is never the sole signal: added/removed diff lines carry `+`/`ÔêÆ` gutter markers and line numbers (not color alone); the stale/warning state pairs the warning color with an icon + heading text; streaming pairs the spinner with a text label.
