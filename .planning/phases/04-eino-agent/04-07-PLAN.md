<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| editor selection (untrusted page content) → agent | The captured selection is page content the user is viewing; it is sent as the `selection` field. It is UNTRUSTED model input but it crosses no NEW boundary — `subscribeAgentChat` / `rewrite` already accept and delimit it server-side (prompts.go delimits untrusted content in the user turn). |
| rewrite result → page write | A rewrite is a PROPOSAL. It crosses into a page write ONLY through the existing human-approved DiffReviewDialog → gated apply/save path. No new write path is introduced. |
| attachment id → agent | The chosen attachment id is sent as `attachment_id`; the server resolves it through the existing read-only `read_attachment_text` tool and the gated summarize-attachment handler. No new access. |

## STRIDE Threat Register

The backend trust boundary is UNCHANGED by this plan (frontend-only wiring; no API-contract change). The load-bearing safeguard — every agent-proposed write passes through the explicit DiffReviewDialog approval and never auto-applies — is preserved exactly.
