# Phase 2 — UI Design Contract

> Visual and interaction contract for Attachments & Text Extraction (ATT-01..09, SEC-02).
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> **This phase EXTENDS the Phase 0/1 design system — it invents no new design language.**
> All values below are the established tokens in `web/src/styles/tokens.css` and the
> shared primitives in `web/src/styles/controls.css`. The executor MUST reference
> `var(--…)` tokens and existing `.btn`/`.input`/`.field` classes — never new hex/px.

---

## Hard Constraints (carried from Phase 0/1)

- **Hidden-Git rule (absolute):** no Git vocabulary anywhere in attachment UI/UX — no
  "commit", "repo", "push", "SHA", "version control", "Git". Mirror `AutosaveStatus`
  ("Saving…", "Saved") which deliberately never says "commit". Replace/upload/remove
  surface plain language ("Replaced", "Removed", "Uploading…").
- **Token-only styling:** components reference `var(--…)` only. No hard-coded hex or px
  except where a token already encodes it.
- **Reuse existing primitives:** `Dialog` (focus-trapped, backdrop-never-confirms),
  `Table`, `.btn`/`.btn-primary`/`.btn-secondary`/`.btn-destructive`/`.btn-ghost`/
  `.btn-ghost-destructive`, `.field`/`.field-error`/`.field-help`, `.input`.
- **Editor-gated mutations:** upload, replace, remove affordances render only when
  `meData?.role === "editor" || "admin"` (same gate as the "Edit page" button in
  `PageView.tsx`). Readers see attachment cards + preview + download, no mutation controls.
- **Light theme only** (`data-color-mode="light"`); tokens are theme-ready but no dark
  theme this phase.

## Spacing Scale

Established 8-point scale (`tokens.css`). Reuse exactly — declare no new spacing.

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| `--space-xs` | 4px | Icon→label gap inside chips/status; card meta inline gaps |
| `--space-sm` | 8px | Gap between action buttons; gap between attachment cards' inner rows |
| `--space-md` | 16px | Default padding inside attachment card; gap between cards in the list; section gap below prose |
| `--space-lg` | 24px | Padding of the dropzone interior; vertical gap above the Attachments section |
| `--space-xl` | 32px | Major break between page body and the Attachments section header |
| `--space-2xl` | 48px | (unused this phase) |
| `--space-3xl` | 64px | (unused this phase) |

**Hit sizes:** interactive controls use `--hit-min-height` (40px); icon-only buttons
(replace / remove on a card) use `--hit-min-icon` (44px) min touch target.

Exceptions: image thumbnail is a fixed **64×64px** square (`--space-3xl`) with
`object-fit: cover`, `--radius-sm` corners. This is a media dimension, not spacing.

| Role | Value | Usage in this phase |
|------|-------|---------------------|

| Secondary (30%) | `--color-surface` `#f6f7f8` | Dropzone idle fill; thumbnail placeholder; card hover; status chip fill |
| Border | `--color-border` `#e3e5e8` | Card border; dropzone dashed border (idle); table/divider lines |
| Muted text | `--color-text-muted` `#6b7280` | Card meta line; "Extracting…"; "No text extracted" body; dropzone hint |
| Accent (10%) | `--color-accent` `#2563eb` | See reserved-for list below |
| Accent surface | `--color-tree-active-bg` `rgba(37,99,235,0.08)` | Dropzone **drag-over** fill (active highlight); accent border on drag-over |
| Destructive | `--color-destructive` `#dc2626` | Validation errors; "Remove" / "Replace" confirm; failed-extraction text |
| Success | `--color-success` `#16a34a` | "Uploaded" / "Saved" transient confirmation; "Text extracted" status icon |
| Warning | `--color-warning` `#d97706` | "No text extracted" status icon (informational, not an error) |

**Accent reserved for (explicit — never "all interactive elements"):**
1. The single primary CTA per surface: the **"Add files"** button (`.btn-primary`) in the
   Attachments section header.
2. The **drag-over** state of the dropzone (accent dashed border + `--color-tree-active-bg` fill).
3. The keyboard `:focus-visible` ring (global 2px accent outline — already established).
4. Download links rendered as text links inherit accent (consistent with prose links).

Everything else (replace, remove, secondary buttons, card chrome) is neutral/ghost.
"Remove" is `.btn-ghost-destructive` (restrained destructive — red text on neutral surface
until confirmed), matching the Phase 1 row-action pattern.

## Dropzone States (AttachmentDropzone)

A single dashed-border drop target. States are mutually exclusive and `aria-live="polite"`.

| State | Border | Fill | Icon | Copy (Body / Label) |
|-------|--------|------|------|---------------------|
| Idle | 1px dashed `--color-border`, `--radius-md` | `--color-surface` | `FilePlus` 16, muted | Body: "Drop a file here or click to browse" · Label (muted): "Up to {max} MB · {allowed types}" |
| Drag-over | 2px dashed `--color-accent` | `--color-tree-active-bg` | `FilePlus` 16, accent | Body (accent): "Drop to upload" |
| Uploading | 1px dashed `--color-border` | `--color-surface` | `Loader2` 16 spinning (`.spinner`/`.autosave-spinner`), muted | Label (muted): "Uploading {filename}…" |
| Client-reject (pre-POST) | 2px solid `--color-destructive` | `--color-surface` | none | `.field-error` (destructive): see copywriting; clears on next interaction |

- Padding: `--space-lg` interior; min-height comfortable (≈120px) so it reads as a target.
- Cursor `pointer` on idle; the whole zone is clickable (opens the file picker).
- Client-side pre-checks (size > `max_upload_mb`, extension not in allow-list) reject
  BEFORE the multipart POST and show the reject state inline — no network round-trip needed
  to tell the user a 200MB file is too big.

## Attachment Card Layout (AttachmentCard)

Horizontal card: `--color-bg` background, 1px `--color-border`, `--radius-md`,
`--shadow-card`, `--space-md` padding, `--space-md` gap between cards.

```
┌───────────────────────────────────────────────────────────────┐
│ [64×64    ]  report-q3.pdf                      [↻] [🗑]        │  ← filename Body/600 + icon actions
│ [thumbnail]  1.4 MB · Jane Doe · 21 Jun 2026                   │  ← meta line, Label/muted
│ [or icon  ]  ◷ Extracting…                                     │  ← ExtractionStatus chip
└───────────────────────────────────────────────────────────────┘
```

- **Left media (64×64, `--radius-sm`):**
  - Image types (png/jpg/svg): `<img>` thumbnail, `object-fit: cover`. Clicking opens the
    inline full preview (see below).
  - Non-image: a `--color-surface` square with a centered lucide type icon
    (`FileText` for pdf/docx/txt, generic file icon otherwise), `--color-text-muted`.
- **Filename:** original name (from meta sidecar), Body size, weight 600. Truncate with
  ellipsis; full name in `title`/tooltip. This is the only place the original name appears
  (never the opaque on-disk id — that stays hidden, like Git).
- **Meta line:** `{humanSize} · {uploaderDisplayName} · {date}`, Label size,
  `--color-text-muted`, `·` separators (matches existing muted-meta style). Date is
  human-friendly ("21 Jun 2026"), never a raw timestamp or SHA.
- **Extraction status chip:** see next section.
- **Actions (right, editors only):** two icon-only `.btn-ghost` buttons,
  `--hit-min-icon` (44px) targets, `--space-sm` gap:
  - **Replace** — `Undo2`/refresh icon, `aria-label="Replace attachment"` → opens
    `ReplaceAttachmentDialog`.
  - **Remove** — `Trash2`, `.btn-ghost-destructive`, `aria-label="Remove attachment"` →
    opens `RemoveAttachmentDialog`.
  - Readers see neither; everyone sees a **Download** affordance (text link or `.btn-ghost`)
    that hits the download endpoint (original bytes; risky types via `Content-Disposition: attachment`).

## Image Preview Presentation

- Thumbnail on the card is always a 64×64 `<img>` for png/jpg/svg.
- Clicking the thumbnail opens an **inline full-size preview** in the existing `Dialog`
  (focus-trapped, Esc/backdrop closes — backdrop never mutates): `<img>` constrained to
  `max-width: 100%` and viewport-bounded `max-height`, centered on `--color-bg`, with the
  filename as the dialog title.
- SVG is shown via `<img src>` only (never inlined / never `dangerouslySetInnerHTML`) so it
  cannot execute script — matches the Phase 1 stored-XSS guard (raw-HTML plugin OFF).
- Non-image, non-previewable types: no preview; the type-icon card + Download only.

## Extraction Status Indicator (ExtractionStatus)

Inline chip on the card, driven by SSE, **modeled exactly on `AutosaveStatus`** (same
`.autosave-status` flex/gap/Label-size structure, `aria-live="polite"`, lucide icons at 14).
Four mutually-exclusive states. **No Git/processing-internals vocabulary.**

| State | Icon (14) | Color | Copy |
|-------|-----------|-------|------|
| Extracting | `Loader2` spinning (`.autosave-spinner`) | `--color-text-muted` | "Extracting text…" |
| Extracted | `Check` | `--color-success` | "Text extracted" |
| No text extracted | `FileText` (or info icon) | `--color-warning` | "No text extracted" |
| Failed | (alert icon) | `--color-destructive` | "Couldn't extract text" |

- **"No text extracted"** is explicit and NON-alarming (warning amber, not red): it is the
  legitimate result for scanned/image-only PDFs, per CONTEXT (text-layer only). Add a muted
  Label sub-note: "This file has no readable text layer (e.g. a scanned PDF)."
- Status applies only to extractable types (pdf/docx/txt). Image/other types show **no**
  extraction chip at all.
- The chip updates live from the SSE stream; on disconnect it falls back to the
  last-known state (no error flash on a dropped stream).

## Accessibility

- All icon-only buttons carry an `aria-label`; lucide icons are `aria-hidden="true"`.
- Dropzone announces state via `aria-live="polite"`; the click target is keyboard-focusable
  and operable with Enter/Space (react-dropzone provides this).
- ExtractionStatus uses `aria-live="polite"` so the live SSE transition is announced once.
- Preview dialog inherits `Dialog`'s focus trap + Esc + focus-restore.
- Validation errors use `.field-error` and are associated with the dropzone for SR users.
- Focus ring is the global 2px accent `:focus-visible` outline.
