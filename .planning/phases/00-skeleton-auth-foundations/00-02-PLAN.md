<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP path param / handler -> filesystem | User-facing path strings (`runbooks/x.md`) are joined to the repo root — the primary traversal surface |
| Filesystem symlinks -> repo root | A symlink inside the repo can point outside it (symlink escape) |
| Multiple writers (handlers + job worker + bootstrap) -> Git index | Concurrent `git` invocations contend on `.git/index.lock` |
| Process crash / restart -> Git working tree | A killed mid-commit process leaves a stale lock / dirty index |
| Optional Git remote -> local repo | A diverged remote could silently merge/clobber local content on pull |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-00.02-01 | Tampering / EoP | `internal/repo.Resolve` | mitigate | Single safe-path resolver rejects `..`/absolute/encoded/NUL after decoding, then `EvalSymlinks` + boundary prefix check + `os.Root`; fuzz-tested; SEC-01. No other code constructs repo paths. |
| T-00.02-02 | Tampering | symlink escape | mitigate | Resolve the FINAL real path (`EvalSymlinks`) and assert it is within the canonical root — lexical cleaning alone is insufficient (Pitfall 5). |
| T-00.02-03 | Denial of service | concurrent git -> `index.lock` | mitigate | Single-writer model: all git ops serialized through one worker goroutine; handlers/bootstrap enqueue intents, never run git concurrently (Pitfall 2). |
| T-00.02-04 | Denial of service / availability | stale `.git/index.lock` after crash | mitigate | Startup self-heal detects and clears a stale lock only when no live git process holds it, then `git status`/`fsck --quick`; surfaces in repo-health (SPEC §6.6). |
| T-00.02-05 | Tampering / data loss | diverged Git remote on pull | mitigate | Fast-forward-only pull; on divergence DO NOT merge — set diverged flag, surface warning banner, skip seeding (D-09/D-12). Push deferred to Phase 1. |
| T-00.02-06 | Tampering (command injection) | git CLI shell-out | mitigate | `exec.Command` with explicit arg slices (no `sh -c`/string concat), exit-code checked; user input never interpolated into a shell string. |
| T-00.02-07 | Repudiation | commit authorship | mitigate | Every commit carries `User`/`Action`/`Source` metadata (SPEC §14.2); the seed commit is authored with the admin identity. |
| T-00.02-SC | Tampering (supply chain) | git binary dependency | accept | Shell-out to the host `git` binary is a LOCKED decision (CLAUDE.md); the binary is part of the deploy trust boundary (same as the OS). No new package-manager installs in this plan beyond locked Go modules. |
</threat_model>
