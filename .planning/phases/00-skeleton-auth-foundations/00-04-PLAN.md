<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Application events -> audit store | Audit must record who did what without leaking secrets |
| Config file / env -> running process | The resolved LLM API key and session secrets must never reach logs |
| Container/host -> running process | The deployed process should run as a non-root user with a scoped data directory |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-00.04-01 | Repudiation | key actions without a trail | mitigate | `audit.Record` writes a SQLite mirror row AND a structured slog line for login/logout/bootstrap/seed/user-management/config changes, capturing actor + target; SEC-05. |
| T-00.04-02 | Information disclosure | secrets in audit / logs | mitigate | Audit never records password plaintext or session tokens; config never logs the resolved `api_key_env` value. |
| T-00.04-03 | Tampering / availability | audit failure breaks auth | mitigate | `Record` is non-fatal — a DB write error is logged but does not propagate to fail the user request. |
| T-00.04-04 | Elevation of privilege | container runs as root | mitigate | systemd unit runs as non-root `okf` user; Dockerfile final stage runs as a non-root user (distroless/scratch). |
| T-00.04-05 | Information disclosure | env var exposure in image | mitigate | API key is supplied via env (`OKF_LLM_API_KEY`) at runtime, not baked into the image; never logged. |
| T-00.04-SC | Tampering (supply chain) | Docker base images + npm ci | mitigate | Pin base images (golang:1.26, a pinned node, distroless); `npm ci` against the committed lockfile; all Go/npm deps are the LOCKED set from CLAUDE.md/STACK.md. Any new dependency triggers a blocking legitimacy checkpoint before install. |
</threat_model>
